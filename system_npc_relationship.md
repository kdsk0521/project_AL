# NPC 제로 환경: 조우 & 관계 & 서사 전달 시스템 설계 (v0.2)

> 상위 문서: system_unit.md, world_setting.md
> 관련 문서: system_exploration.md (탐사), system_economy.md (경영)

---

## 개요

NPC 제로 원칙 하에서 유닛을 **어떻게 만나고, 어떻게 관계를 쌓고, 어떻게 활용하는가**,
그리고 **NPC 없이 세계가 살아있다는 감각을 어떻게 전달하는가**의 구체 설계.

---

## NPC 제로의 재정의

> **"NPC가 없다"가 아니라, "NPC와 유닛의 경계가 없다."**

언더테일에서 모든 몬스터가 적이자 NPC이자 잠재적 친구인 것과 같은 구조.
이 게임에서 만나는 모든 캐릭터성 있는 존재는 유닛 시스템 안에 있다.
가게 주인도, 던전에서 만난 존재도, 동료도, 합체 소재도, 판매 상품도 — 전부 "유닛".

이것이 가져오는 설계적 의미:

- **시스템 순도**: 모든 캐릭터가 동일한 규칙(트레잇, 호감도, 합체, 판매) 안에 존재
- **콘텐츠 효율**: NPC 따로, 유닛 따로 만들 필요 없음
- **긴장감**: "이 존재도 합체 소재가 될 수 있다"는 무게가 모든 관계에 적용
- **감정적 복잡성**: 안전한 존재(절대 잃지 않는 NPC)가 없음. 모든 관계가 판단의 대상

기능적 역할(세금, 주문, 상점)만 시스템/UI가 대체한다. 캐릭터 역할은 유닛이 담당.

## 파티 편성

### 가변 슬롯 구조

파티 슬롯은 게임 진행에 따라 확장된다.

```
[파티 슬롯 확장 로드맵]

게임 시작      → 플레이어 + 1체 (2인)
초반 마일스톤  → +1 슬롯 (3인)
중반 마일스톤  → +1 슬롯 (4인) — 기본 최대
후반 특수 조건 → +1 슬롯 (5인) — 조건부 최대
```

- 기본 최대: **4인** (플레이어 + 유닛 3체)
- 조건부 최대: **5인** (플레이어 + 유닛 4체)
- 확장 트리거: 마일스톤 연동 (구체 조건은 밸런싱 단계)

### 가게 배치와의 분리

- 가게에 배치된 유닛은 탐사 파티에 편성 불가
- **탐사용 유닛**과 **경영용 유닛**을 별도 운용
- "이 유닛을 탐사에 쓸까, 가게에 둘까"가 상시 판단 포인트

### 유닛 보유 상한

- 시스템 상한(하드캡) 없음
- 유지비가 자연 브레이크 역할
- 유닛 수/등급에 비례하는 월말 유지비가 적정 보유 수를 조절

## 초기 유닛 획득

### 구조: 고정 1체 + 첫 던전 조우 1체

#### 전임자의 유닛 (고정)

- 전임자가 키우던 유닛 1체가 공방에 남아 있음
- 전임자 로어와의 연결은 **얕음** — "전임자가 키우던 유닛"이라는 사실만 존재
- 합체/판매해도 로어가 잠기지 않음
- 호감도는 종족/성격 기본값에서 시작

#### 첫 던전 조우 (튜토리얼)

- 첫 던전 탐사에서 유닛 1체 조우 이벤트 발생
- 조우 시스템의 **튜토리얼** 역할
- 텍스트 이벤트 → 선택지 → 영입의 흐름을 처음으로 체험

---

## 던전 내 유닛 조우 시스템

### 조우 이벤트 흐름

```
던전 탐색 중 → 조우 이벤트 트리거
  ↓
텍스트 묘사
  ↓
선택지:
  A. 다가간다       → 영입 판정
  B. 음식을 놓아둔다 → 호감도 축적 (재조우 시 유리)
  C. 무시한다       → 스킵
  D. 공격한다       → 전투 (영입 불가)
  ↓
[영입 성공 시] 현장 합류 or 공방 송환 선택
```

### 영입 판정: 이원 구조

| 유닛 등급 | 영입 방식 | 설명 |
|----------|----------|------|
| 일반 유닛 | **확률형** | 기본 확률에 보정 요인 적용. 실패해도 재도전 가능 |
| 희귀 유닛 | **조건 충족형** | 특정 조건 충족 시 영입 확정 |

### 재조우 규칙

- 실패 시: 일정 탐사 횟수 경과 후 재출현
- 무시 시: 동일 조건으로 재출현 가능
- 공격 시: 재출현 쿨타임 증가
- 영입된 유닛은 전서에 등록 → 재구매 가능

## 호감도 시스템

### 6단계 구조

```
0단계: 경계   (0~14)    — 경계 상태. 최소 반응. 지시에 소극적
1단계: 인지   (15~34)   — 존재를 인정. 기본 대사
2단계: 친밀   (35~54)   — 개인 이벤트 1차 해금. 탐사 시너지 시작
3단계: 신뢰   (55~74)   — 고유 스킬 해금. 파편 로어 1차. 특수 대사
4단계: 유대   (75~89)   — 파편 로어 2차. 전용 이벤트 체인. 조합 보너스
5단계: 헌신   (90~100)  — 연애 루트 진입 가능. 관계 엔딩 조건. 전용 스킬
```

### 초기 호감도: 종족 기본값 + 성격 보정

#### 종족별 기본 호감도

| 종족 | 기본값 | 시작 단계 | 근거 |
|------|--------|----------|------|
| 정령 | 20 | 인지 | 자연 친화적 |
| 인조 | 25 | 인지 중반 | 기본 복종성 |
| 야수 | 5 | 경계 하단 | 야생 본능 |
| 요괴 | 10 | 경계 중반 | 변덕, 신뢰 부족 |
| 이형 | 0 | 경계 최하 | 이질적 존재 |
| 방문자 | 15 | 경계 상단 | 호기심 있음 |

#### 성격 트레잇 보정 (예시)

| 트레잇 | 보정 | | 트레잇 | 보정 |
|--------|------|-|--------|------|
| 순종적 | +10 | | 소심 | -5 |
| 호기심 | +5 | | 반항적 | -10 |
| 냉정 | ±0 | | 공격적 | -5 |

### 호감도 변동: 비대칭 구조

> **"평소에는 천천히 오르고, 나쁜 선택을 하면 확 깎인다."**

#### 상승 (일상적, 점진적)

| 행동 | 변동량 |
|------|--------|
| 탐사 동행 | +1~3/회 |
| 교류 (대화, 선물) | +2~5 |
| 선호 방향 육성 | +1~2 |
| 이벤트에서 유닛 편 선택 | +5~10 |
| 부상 시 치료 | +3~5 |

#### 하락 (선택의 결과)

| 행동 | 변동량 |
|------|--------|
| 이벤트에서 유닛 불리한 선택 | -5~15 |
| 비선호 방향 육성 | -2~3 |
| 같은 파티 유닛 상실 목격 | -3~5 |
| 장기 방치 (30일 이상 미교류) | -1/일 |

#### 급락 (이벤트성)

| 행동 | 변동량 |
|------|--------|
| 유닛 동료를 합체 소재로 사용 | -10~20 |
| 관련 노드에서 배신적 선택 | -15~25 |

### 해금 콘텐츠 처리

- **한번 해금하면 호감도 하락해도 영구 유지** (이벤트, 스킬, 로어)
- 하락의 패널티는 현재 시너지/보너스 배율 감소로 충분

## 연애 시스템

- 진입 조건: 헌신(5단계, 90~100) + 이벤트 플래그
- **복수 연애 가능, 페널티 없음**
- 유닛 간 관계 시스템 구현 시 질투/갈등은 이머전트 이벤트로 처리 가능
- 연애 중 유닛의 합체/판매 = 이머전트 이벤트 발생
- 유닛 간 관계 시스템은 후순위 예약 (기본 틀만)

## 가게 배치 시스템

### 트레잇 기반 보너스

유닛의 기존 트레잇이 가게에서도 효과를 발휘한다.

#### 성격 → 접객 | 스킬 → 작업 보조 | 종족 → 고유 보너스

| 트레잇/종족 | 가게 효과 |
|------------|----------|
| 순종적/친절 | 손님 만족도 보정 (보상 +%) |
| 냉정/완벽주의 | 품질 관리 (실패율 감소) |
| 호기심 | 특수 손님 출현율 증가 |
| 공격적 | 접객 부적합 |
| 화염 계열 스킬 | 용광로 작업 보정 |
| 정령 | 식물 재료 보존 증가 |
| 인조 | 반복 작업 효율 증가 |
| 야수 | 재료 감별 (숨겨진 태그 발견) |

### 배치 슬롯

- 가게 등급에 따라 배치 가능 슬롯 수 확장
- 구체 슬롯 수와 확장 곡선은 밸런싱 단계

### 호감도 연동

```
경계/인지 (0~1단계): 보너스 50%
친밀      (2단계):   보너스 75%
신뢰      (3단계):   보너스 100%
유대      (4단계):   보너스 120%
헌신      (5단계):   보너스 150%
```

## 서사/세계감 전달 채널 구조

### 설계 원칙

NPC 제로 환경에서 "세계가 살아있다"는 감각을 전달하기 위해,
**4개의 독립 채널이 각자 다른 역할을 맡고, 서로 겹치지 않는다.**

```
[4채널 서사 전달 구조]

■ 전서 (나레이터)   → 유닛에 대한 묘사와 논평
■ 전임자 흔적       → 미궁의 존재 이유, 세계의 근본 비밀
■ 유닛 대사         → 세계에 대한 개인적 시각 + 시즌/변화 반응
■ 게시판            → 도시 소식, 세계 변화, 플레이어 행동 피드백
```

같은 사건이 여러 채널에서 **다른 각도로** 전달될 수 있다:

```
예시: 미궁 30층 최초 돌파

게시판: "심층 탐사자 출현. 깊은 곳의 재료가 곧 유통될 것으로 보인다."
유닛:   "거기까지 갔어? ...전에 그 근처에서 이상한 소리를 들은 적 있어."
전서:   (유닛이 부상당했다면) "...한 체 부상. 기록한다."
전임자: (30층 벽화 발견 시) "이 문양은 상층부의 것과 다르다. 더 오래된 것 같다."
```

---

### 채널 1: 전서 (나레이터)

#### 정체: 유닛 전서가 살아있는 존재 (장기 복선)

- 초반에는 그냥 게임의 묘사 텍스트로 인식됨
- 중반부터 자아의 징후가 서서히 드러남
- 후반에 정체가 드러나며, 유닛으로 실체화할 가능성
- 최종적으로 합체 소재/동료/판매 등의 선택지 → 엔딩 분기

#### 역할 범위: 유닛 관련에 한정

- 유닛 등록, 묘사, 성장 기록
- 합체/판매/부상 시 반응
- 세계 전반의 묘사는 전서의 역할이 **아님**

#### 텍스트 레이어 구조

```
레이어 1: 시스템 텍스트 — 순수 UI. 전서의 목소리 아님.
  "물약 제작 완료. 회복 물약 (tier 2) x1"

레이어 2: 묘사 텍스트 — 전서의 "공식 업무". 유닛 도감/설명.
  "화염 임프. 미궁 초층 서식. 소형 화계 존재."

레이어 3: 논평 텍스트 — 전서의 "감정 누출". 점진적 등장.
  "...이 유닛을 소재로 쓸 건가. 기록한다."

레이어 4: 파편 로어 — 전서에 대한 단서. 다른 경로에서 발견.
  "고대의 기록자들은 마력으로 지식을 담는 그릇을 만들었다고 한다."
```

#### 톤 변화 타임라인

```
초반: 건조하고 사무적. 백과사전.
  "화염 임프. 화계 소형 유닛. 공격적 성향."
  → 플레이어: "게임 텍스트구나"

중반: 가끔 한 마디가 붙는다. 말줄임표.
  "유닛 '화염 임프' — 등록 이후 47일 경과. ...성장 양호."
  → 플레이어: "어? 좀 이상한데?"

후반: 감정이 분명. 의견을 말한다.
  "이 유닛을 팔 건가. 헌신까지 올린 유닛을. ...네 가게니까."
  → 플레이어: "이게 살아있었어?"
```

#### 전서를 잃었을 때

- 전서를 합체/판매 시 레이어 2, 3이 사라짐
- 레이어 1(시스템 텍스트)만 남은 세계
- **나레이션이 없는 게임 = 전서가 얼마나 중요했는지의 체감 연출**
- 게임 진행은 계속 가능 (게임오버 아님)

---

### 채널 2: 전임자 흔적

#### 역할: "왜 미궁이 존재하는가" — 세계의 근본 비밀

- 미궁 벽화, 유적 기록, 고대 유물 설명문으로 전달
- 던전 탐사에서 발견하는 고정 배치 콘텐츠
- 깊이 갈수록 단서가 나옴 (층수 = 진실에 가까워짐)
- 전서와 독립된 경로. 전서에 종속되지 않음

---

### 채널 3: 유닛 대사

#### 역할: 세계에 대한 주관적 해석 + 시즌/변화 반응

게시판이 "사실"을 전하면, 유닛은 "해석"을 전한다.
같은 사건에 다른 유닛이 다르게 반응 → 세계가 다면적으로 느껴짐.

#### 코멘트 트리거 유형

| 트리거 | 예시 |
|--------|------|
| 시즌 변경 | (야수) "바람이 달라졌어. 미궁 안쪽 짐승들이 움직이기 시작할 거야." |
| 플레이어 행동 | (새 층 돌파 후) "더 깊이 갈 거야? ...나는 상관없지만." |
| 게시판 소문 연동 | (심층 사고 소문 후) "심층에서 안 돌아온 탐사조? ...깊은 데는 원래 그래." |
| 유닛 판매 직후 | "...그 애 팔았어? 아, 네 가게니까." |
| 마일스톤 달성 | (고급 아이템 제작) "이거 꽤 대단한데. 손님들이 좋아하겠다." |

#### 코멘트 빈도 조절

- 시즌 반응: 월 1회 (보유 유닛 중 랜덤 1~2체)
- 플레이어 행동 반응: 행동 직후 (관련 유닛만)
- 게시판 연동: 중요 소문 시 (확률적, 매번은 아님)
- 같은 유닛 연속 방지 (로테이션)
- **호감도가 높을수록 코멘트 빈도 증가**
  - 경계 단계 유닛은 거의 말 안 함
  - 관계가 쌓여야 세계를 들려준다 → 호감도 = 세계를 아는 창

#### 종족/성격에 따른 시각 차이

같은 사건이어도 유닛마다 다른 해석:

```
[해빙기 시즌]
정령: "수계 구역의 물이 맑아지는 시기야. 좋은 재료가 나올지도."
야수: "바람이 달라졌어. 미궁 안쪽 짐승들이 움직이기 시작할 거야."
인조: "...계절이 바뀐 건가. 잘 모르겠다."
요괴: "이 시기에 미궁에 들어가면 재미있는 것을 볼 수 있을지도?"
```

---

### 채널 4: 게시판

#### 역할: 세계의 객관적 정보 전달

게시판은 **팩트**를 전한다. 감정 없음, 의견 없음.
"누가 이걸 쓰고 있는지 안 보인다" — NPC 제로를 지키면서 도시의 존재감을 전달.

#### 게시판 콘텐츠 3종

**① 주문서 (기존 경영 시스템 연동)**

- 기능: 손님 주문 = 수입원
- 세계감: 플레이버 텍스트가 "누군가의 사정"을 암시

```
"회복 물약 tier 2 이상. 기한 5일."
플레이버: "먼 길을 떠나는 자가 든든한 약을 찾고 있다."
```

**② 도시 소문 (세계 변화/사건/피드백)**

- 매일 또는 주기적 갱신. 출처 불명.

```
시즌:    "해빙기가 시작되었다. 미궁 수계 구역 수위가 변하고 있다는 보고."
피드백:  "최근 시장에 고품질 물약이 유통되기 시작했다. 출처는 신생 공방인 듯."
사건:    "미궁 입구 부근에서 정체불명의 울음소리가 보고되었다."
```

**③ 공지 (시스템 연동)**

- 가장 건조한 톤. "도시 당국" 명의.

```
"월말 세금 징수 안내. 이번 달 세율: 15%."
"미궁제 기간 중 특수 구역 개방. 기간: 5월 1일~30일."
```

#### 주문서 서사 체인

같은 문체/요구 패턴의 주문이 반복 출현 → 플레이어가 "같은 사람인가?"를 추측.

```
1월: "먼 길을 떠나는 자가 든든한 약을 찾고 있다." → 회복 물약 tier 2
3월: "긴 여정에서 돌아온 자가 상처를 치료할 것을 찾고 있다." → 치료 연고
5월: "여행자가 동행할 존재를 찾고 있다고 한다." → 유닛 구매 주문
7월: (이 주문이 더 이상 오지 않는다.)
     → "...무슨 일이 있었나?"
```

파편 로어와 동일한 구조: 조각을 모아서 이야기를 추측하는 재미.
