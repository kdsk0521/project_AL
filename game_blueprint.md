# 텍스트 MUD 게임 블루프린트 (v0.4)

## 개요

던전 탐사 + 연금술 조합 + 경영 시뮬레이션을 결합한 텍스트 기반 MUD 게임.
최소한의 UI로 텍스트 중심 플레이를 지향하며, ERA 스타일의 스탯 기반 시뮬레이션을 참고한다.

---

## 설계 철학

> **"콘텐츠는 유한해도, 경험은 무한하다."**

이 게임의 모든 설계 판단은 이 원칙에 기반한다.

- **콘텐츠(아이템, 유닛, 로어, 엔딩)는 유한하다.** 다 모을 수 있고, 다 볼 수 있다. 그것이 목표가 되어도 좋다.
- **그러나 경험은 무한하다.** 같은 재료, 같은 유닛, 같은 시스템이라도 조합 순서, 플레이어의 선택, 시스템 간 충돌 타이밍에 따라 매번 다른 이야기가 만들어진다.
- **"다 안다"가 성립하지 않는 깊이.** 시스템 요소들의 조합 공간이 충분히 넓어서, 모든 것을 파악한 플레이어도 "이번엔 다르게 해보면?"이라는 질문을 스스로에게 던지게 된다.
- **자기 규칙 플레이를 허용하는 유연성.** 시스템은 플레이어가 스스로 도전 조건을 설정할 수 있을 만큼 유연해야 한다. 개발자가 만들지 않은 놀이를 플레이어가 발명하는 것이 이상적인 상태.
- **억압이 있어서 자유가 있다.** 자유도는 처음부터 전부 열리지 않는다. 진행 상태에 따라 가용 범위가 점진적으로 확장되어야 한다. 초보자가 압도되지 않으면서 숙련자가 깊이를 누릴 수 있는 단계적 개방 설계.

설계 시 자문: *"이 시스템은 100시간 플레이한 사람에게도 새로운 경험을 줄 수 있는가?"*

### 코어 문법: 모든 것은 조합이다

이 게임의 모든 시스템은 **"재료를 모아서 조합한다"**는 하나의 문법으로 작동한다.

```
재료 태그 조합     → 아이템
가공 체인          → 특성 변환/등급 상승
트레잇 조합        → 합성 트레잇 발현
유닛 합체          → 새로운 유닛 (트레잇 계승)
이벤트 조건 조합   → 이머전트 이벤트 발생
파편 로어 수집     → 세계관 조립
```

플레이어가 한 시스템의 문법을 이해하면 다른 시스템도 자연스럽게 이해한다.
"이것과 저것을 합치면 무엇이 되는가?"가 게임 전체를 관통하는 질문.

## 게임 구조

### 플레이어 포지션
- 연금술사 겸 도시 복원자 (공방 운영자, 탐사자, 조합사를 겸하는 종합 포지션)
- 구체적인 설정은 세계관 확정 시 결정

### 핵심 루프
```
던전 탐사 → 재료 수집 + 유닛 획득 → 조합/제작 + 육성 → 도시 복원/유닛 납품 → 더 깊은 던전
     ↑                                                    |
     └────────────────────────────────────────────────────┘
            ↕ [이벤트 간섭] ↕ [로어 발견] ↕ [노드 분기]
```

### 진행 구조
- **던전**: 무한 확장. 층수/구역이 계속 늘어나며 난이도 스케일링
- **엔딩**: 업적형 수집 엔딩. 특정 조건 달성 시 엔딩 장면이 해금됨
- **엔딩 이후**: 계속 플레이 가능, 미해금 엔딩 수집이 반복 동기
- **"더 할 게 없다"는 느낌이 없을 것**: 시스템이 이야기를 생성하고, 이야기가 시스템을 열어주는 순환

## 코어 시스템 (4축)

### 1. 탐사
- 던전 진입 → 층/구역 이동 → 채집, 전투, 이벤트 → 귀환
- 유닛(동료)을 편성하여 동행
- 무한 확장 구조: 층수 증가에 따른 난이도/보상 스케일링
- 랜덤 이벤트, 보스, 숨겨진 구역 등
- 탐사 결과가 경영/관계/조합에 파급 (→ 시스템 간섭)

### 2. 조합/제작
- **아이템 합성**: 태그 합산 방식. 4축(기능+재료성질+원소+형태) 태그 조합이 결과물 결정
- **가공**: 쌍원소 가공 — 설비마다 주입+해방 두 원소를 동시 처리. 주입=기능변환, 해방=형태변환+원소제거
- **유닛 합체**: 메가텐식 종족×종족 테이블 기반. 레벨 평균으로 결과 결정
- **특수 레시피**: 지정 조합에서만 나오는 예외 결과물 (기초 9종 + 숨겨진 18종 확정)
- 조합 결과물은 **판매하지 않음** — 유닛에게 주거나 탐사에 사용하는 상호작용 수단
- 조합 결과물에 로어 텍스트 부착 (→ 파편 로어)

### 3. 경영
- 비어있는 미궁도시를 유닛으로 채워가는 **도시 복원 시뮬레이션**
- 통화 = 전서 포인트(영혼력). 유닛 납품이 유일한 정규 수입
- 아이템 판매 없음 — 조합물은 유닛에게 주는 투자/탐사 소비재
- 도시 시설에 유닛 배치 → 기초 재료 자동 수급, 기능 해금
- 설비 업그레이드 (작업대, 가공 설비 등)
- 유닛의 4방향 용도: 납품(수입) / 배치(투자) / 합체(강화) / 탐사(전력)
- 경영 상태가 유닛 관계/탐사 가용 범위에 영향 (→ 시스템 간섭)

### 4. 관계
- 유닛(NPC/동료)과의 호감도 시스템
- 호감도 단계별 이벤트 트리거
- 연애 루트 포함 ("안 해도 되지만 있으면 하게 되는" 서브 시스템)
- **다른 시스템과의 연결 고리**:
  - 탐사 동행 시 호감도 상승
  - 높은 호감도 → 조합 보너스, 탐사 버프 등
  - 특정 관계 조건이 엔딩/노드 해금에 관여
  - 유닛 간 관계가 이벤트 발생 조건에 영향 (→ 시스템 간섭)

## 내러티브 아키텍처

이 게임은 개발자가 스토리를 전달하는 것이 아니라, 4개의 내러티브 레이어가 겹쳐져
플레이어 각자의 이야기가 만들어지는 구조를 지향한다.

### 레이어 1: 이머전트 이벤트 (림월드식 — 이야기의 뼈대)

핵심 원칙: **시스템 간 간섭이 사건을 만든다.**

개발자는 스토리를 쓰지 않는다. 대신 시스템이 충돌할 조건을 설계한다.
플레이어가 겪는 "이야기"는 시스템 상호작용에서 자연 발생한다.

#### 시스템 간섭 매트릭스
```
         탐사        조합        경영        관계
탐사      --      재료 수급    비용/보상   동행/부상
조합    장비강화     --       유닛 투자   합체 소재
경영    탐사 투자  설비 품질     --       고용/해고
관계    전투 시너지 합성 보너스  충성/이탈    --
```

- 각 칸이 "간섭 지점"이며, 간섭이 발생할 때마다 이벤트 후보가 트리거됨
- 예시 시나리오:
  - 영혼력 부족(경영) → 유닛 납품 압박(관계) → 애착 유닛을 납품할 것인가(선택)
  - 탐사 중 유닛 부상(탐사) → 희귀 재료로 치료약 제작(조합) → 재료 소진으로 주문 납기 실패(경영)
  - 호감도 MAX 유닛이 합체 최적 소재(관계×조합) → 합체시킬 것인가 남길 것인가(선택)

#### 이벤트 풀 구조
```json
{
  "event_id": "EVT_001",
  "trigger": {
    "type": "system_clash",
    "conditions": {
      "economy.funds": {"op": "<=", "value": 100},
      "party.size": {"op": ">=", "value": 3}
    }
  },
  "description": "공방의 영혼력이 바닥나고 있다. 누군가를 전서에 보내야 할지도 모른다.",
  "choices": ["유닛 납품", "도시 시설 축소", "긴급 탐사"],
  "consequences": { ... }
}
```

- 이벤트는 시스템 상태 조합에 의해 자동 트리거
- 플레이어 선택에 따라 후속 상태가 변경되고, 그 변경이 또 다른 이벤트를 유발
- **이벤트 자체가 스토리의 문장이 되고, 연쇄가 문단이 되고, 플레이 전체가 그 플레이어만의 이야기가 된다**

### 레이어 2: 파편 로어 (프롬소프트웨어식 — 세계의 깊이)

핵심 원칙: **세계관은 플레이어가 직접 조립한다.**

스토리를 직접 전달하지 않는다. 대신 세계의 조각을 곳곳에 숨겨놓고,
플레이어가 수집·조합하여 전체 그림을 맞추도록 한다.

#### 로어가 숨겨지는 장소
- **아이템 설명문**: 모든 조합 결과물/재료에 1~2줄의 로어 텍스트
- **던전 텍스트**: 특정 층/구역 도달 시 환경 묘사에 담긴 단서
- **유닛 대사**: 호감도 단계별로 해금되는 과거 이야기, 세계 정보
- **특수 이벤트**: 이머전트 이벤트 중 특정 조건에서만 드러나는 추가 텍스트

#### 로어 도감
- 수집한 로어 파편을 카테고리별로 열람 가능
- 빈칸이 보이되, 내용은 안 보임 (수집 동기)
- 파편 간 연결 관계는 플레이어가 스스로 유추 (명시적 연결 없음)
- 특정 파편 조합이 숨겨진 레시피/던전/엔딩의 힌트가 됨

#### 데이터 구조
```json
{
  "lore_id": "LORE_042",
  "category": "world_history",
  "text": "이 금속은 '심층'에서만 채굴된다. 마지막으로 심층에 닿은 자는 돌아오지 못했다고 한다.",
  "source_type": "item_desc",
  "source_id": "ITEM_DEEP_ORE",
  "hidden_link": ["LORE_078", "LORE_103"]
}
```

### 레이어 3: 마일스톤 (방치형/업적식 — 성장의 리듬)

핵심 원칙: **진행할수록 새로운 층이 열린다.**

무한 루프 속에서 일정 구간마다 "무언가 달라졌다"를 느끼게 하는 장치.
이야기의 끝이 아니라 이야기의 다음 챕터를 여는 기능.

#### 마일스톤 유형
- **경영 마일스톤**: 공방 등급 상승, 도시 시설 확장, 특정 도시 복원율 달성
  → 새로운 설비, 도시 기능, 전서 유닛 풀 확장
- **탐사 마일스톤**: 던전 특정 층 최초 돌파, 보스 격파
  → 새로운 던전 구역, 재료 티어, 이벤트 풀 추가
- **조합 마일스톤**: 도감 특정 비율 달성, 전설급 아이템 최초 제작
  → 상위 합성 테이블 해금, 특수 레시피 출현
- **관계 마일스톤**: 호감도 특정 단계, 연애 루트 진입
  → 전용 이벤트 체인, 동행 스킬 해금

#### 마일스톤과 다른 레이어의 연결
- 마일스톤 달성 → 새로운 이벤트 풀이 이머전트 시스템에 추가됨
- 마일스톤 달성 → 새로운 파편 로어가 출현 가능한 상태가 됨
- 마일스톤 달성 → 새로운 노드 분기가 개방됨

---

### 레이어 4: 노드 분기 (비컴휴먼식 — 선택의 무게)

핵심 원칙: **선택이 세계를 분기시키고, 분기는 수집 대상이 된다.**

이머전트 이벤트 중 특별히 중요한 것들은 "노드"로 지정된다.
노드에서의 선택은 후속 게임 상태를 크게 바꾸며, 플로우차트에 기록된다.

#### 노드의 특성
- 모든 이벤트가 노드는 아님. 게임 흐름을 유의미하게 바꾸는 핵심 분기점만 노드
- 노드는 마일스톤 달성, 특정 이벤트 연쇄, 또는 파편 로어 수집에 의해 출현
- 한번 선택하면 되돌릴 수 없음 (해당 회차에서)

#### 노드 분기 예시
```
[노드 N-07: 심층의 문]

출현 조건: 던전 50층 돌파(마일스톤) + LORE_042 수집(파편) + 영혼력 5000 이상(경영)

선택지:
  A. 문을 연다 → 심층 던전 해금, 새로운 재료 티어, 위험도 대폭 상승
  B. 봉인한다 → 봉인 보상 획득, 심층 관련 이벤트 체인 개시
  C. 조사만 한다 → 추가 파편 로어 3개 획득, 이후 재선택 기회 (1회)

결과: 선택에 따라 이후 이벤트 풀, 조우 가능 유닛, 가용 재료가 달라짐
```

#### 플로우차트 (수집 요소)
- 플레이어가 거쳐온 노드와 선택이 플로우차트로 기록됨
- 선택하지 않은 분기는 "???"로 표시 → "다음엔 저쪽을 골라봐야지" 동기
- 플로우차트 자체가 수집 콘텐츠 (전체 노드 맵의 몇 %를 탐색했는가)
- 엔딩 수집과 연동: 특정 노드 경로 조합이 엔딩 해금 조건

---

### 4개 레이어의 상호작용 요약

```
[이머전트 이벤트] ← 시스템 간섭이 사건을 만든다 (일상, 항상 발생)
       ↓ 특정 이벤트가 축적/연쇄되면
[마일스톤 달성]   ← 새로운 층이 열린다 (단계적, 비가역)
       ↓ 마일스톤이 조건을 충족하면
[노드 분기 출현]  ← 중요한 선택을 요구한다 (희소, 고임팩트)
       ↓ 노드 선택이 세계 상태를 변경하면
[파편 로어 출현]  ← 세계의 진실이 조금씩 드러난다 (수집, 자발적)
       ↓ 파편이 힌트가 되어 플레이어 행동이 바뀌면
[이머전트 이벤트] ← 새로운 사건이 발생한다
       ...무한 순환
```

- 이 순환이 "끝없는 던전 안에서 끝없는 이야기가 만들어지는" 구조의 핵심
- 개발자는 이야기를 쓰는 것이 아니라, 이야기가 만들어질 조건과 재료를 설계한다

## 유닛 구조 (초안)

유닛 = 전투원 + 관계 대상 + 합체 소재 + 로어 캐리어의 복합 역할

### 핵심 원칙: 같은 유닛이라도 어떻게 키웠느냐에 따라 다른 존재가 된다 (ERA식 육성 분기)

### 스탯 블록
```
[전투]
- HP, 공격, 방어, 속도
- 속성/타입
- 스킬 슬롯

[관계]
- 호감도 (수치)
- 성격 태그
- 이벤트 플래그
- 노드 선택 기억 (해당 유닛과 관련된 노드에서의 선택 이력)

[합체]
- 종족/타입 (합체 테이블 참조용)
- 레벨
- 계승 가능 스킬/특성

[로어]
- 해금 가능 파편 목록 (호감도 단계별)
- 출신/배경 태그 (이벤트 트리거 조건으로 활용)

[육성 이력] ← ERA 참고
- 성장 경로 태그 (어떤 방식으로 육성했는가의 누적 기록)
- 획득 특성 목록 (육성 방향에 따라 분기되는 특성)
  - 전투 특성, 성격 특성, 성인 특성 등 복수 카테고리 포함 예정 (ERA식 조교 특성 계열)
- 경험 이력 (동행 던전, 참여 이벤트, 겪은 노드 선택 등)
- 육성 방향이 영향을 주는 범위:
  → 합체 결과물의 특성 계승 방향
  → 탐사 시 반응/행동 패턴
  → 이벤트 발생 조건 및 선택지 변화
  → 납품/거래 시 가치 산정
  → 관계 이벤트의 톤과 분기
```

## 엔딩 시스템

### 구조
- 업적형 수집 엔딩 (CG 갤러리식)
- 엔딩 번호를 카테고리별로 분류
  - **1-X**: 경영 엔딩 (도시 복원 목표, 공방 최고 등급 등)
  - **2-X**: 관계 엔딩 (호감도 MAX, 연애 루트 완료 등)
  - **3-X**: 탐사 엔딩 (특정 던전 클리어, 희귀 발견 등)
  - **4-X**: 조합 엔딩 (전설급 아이템 제작, 도감 완성 등)
  - **0-X**: 특수/배드 엔딩 — 경영 실패의 다양한 형태 + 개그성 엔딩
    - 0-1: 공방 정지 (영혼력 0 + 회복 불가)
    - 0-2: 도시 붕괴 (도시 시설 연쇄 정지 → 기능 상실)
    - 0-3: 공방 폐쇄 (점검/안전 기준 미통과)
    - 0-4: 유닛 반란 (호감도 관리 실패 → 유닛 이탈/반란)
    - 0-?: 개그 엔딩 (황당한 조합 사고, 예상치 못한 상황 등)
      - 이스터에그: 플레이어 자신을 조합기에 넣으면 발생하는 엔딩
    - 배드엔딩도 수집 대상 → "일부러 보러 가는" 플레이 유도
  - **9-X**: 노드 경로 엔딩 (특정 노드 선택 조합으로만 도달)

### 조건 유형
- **단일 수치 조건**: 도시 복원율 100% 달성, 호감도 MAX 등
- **복합 조건**: 특정 유닛 + 특정 던전 + 특정 아이템 조합
- **노드 경로 조건**: 특정 노드들에서 특정 선택 조합을 했을 때
- **파편 로어 조건**: 특정 로어 파편 세트를 모두 수집했을 때
- **이머전트 조건**: 특정 이벤트 연쇄가 자연 발생했을 때 (숨겨진 엔딩)
- **숨겨진 조건**: 힌트 없이 우연히 달성 (발견의 재미)

### 엔딩 해금 연출
- 조건 충족 시 엔딩 장면(텍스트) 출력 → 엔딩 번호 수집 목록에 등록
- 수집 현황 열람 가능 (해금/미해금 표시)
- 노드 플로우차트와 연동하여 "어떤 경로로 이 엔딩에 도달했는가" 표시

## 수집 요소 총괄

플레이어의 "더 하고 싶다"를 유지하는 수집 시스템 전체 맵.

| 수집 대상      | 공급원            | 열람 방식          | 완성 동기              |
|---------------|------------------|--------------------|-----------------------|
| 엔딩          | 전 시스템         | 엔딩 갤러리        | 전체 엔딩 컴플리트     |

| 아이템 도감    | 조합/제작         | 도감               | 전체 레시피 발견       |
| 유닛 도감      | 합체/이벤트       | 도감               | 전체 유닛 해금         |
