> 상위 문서: game_blueprint.md (코어 시스템 4축: 관계 + 유닛 구조)
> 참조: system_unit_sigil.md (인(印) 체계 상세), system_crafting.md (태그 체계), system_economy.md (납품/영혼력), world_setting.md (세계관)

---

## 개요

유닛은 이 게임에서 가장 복합적인 존재.
하나의 유닛이 **전투원 + 관계 대상 + 합체 소재 + 로어 캐리어 + 도시 운영자 + 전서 납품 자원**의 역할을 동시에 가진다.

> **"같은 유닛이라도 어떻게 키웠느냐에 따라 다른 존재가 된다."** (ERA 참고)

---

## 설계 원칙

- 유닛은 스탯 덩어리가 아니라 **감정이 붙는 대상**
- 육성 과정이 곧 분기 → 같은 종류의 유닛이라도 플레이어마다 다른 결과
- 합체 소재로 쓸 때, 팔 때 **고민이 생기는** 구조 (ERA식 감정 비용)
- 관계(호감도, 연애)는 "안 해도 되지만 있으면 하게 되는" 서브 시스템

## 유닛 획득 경로

유닛을 얻는 방법은 3가지.

### 1. 전서 구매 (유닛 전서)

페르소나 악마전서와 동일한 구조.

- **기초 풀**: 게임 시작 시 구매 가능한 기본 유닛 몇 종
- **해금 확장**: 마일스톤/엔딩 달성 시 새로운 유닛이 전서에 추가
- **전서 등록**: 한번 합체로 만들었거나 이벤트로 영입한 유닛은 전서에 등록 → 이후 재구매 가능
- 재구매 시 **기본 상태로 초기화** (육성 이력/호감도는 리셋)
- 전서 등록률 자체가 수집 요소 (유닛 도감과 연동)

```json
{
  "compendium_entry": "UNIT_FIRE_IMP",
  "name": "화염 임프",
  "base_price": 500,
  "unlock_type": "default",
  "registered": true,
  "first_obtained": "합체"
}
```

#### 전서의 역할

- **합체 부담 경감**: 소재로 써도 다시 살 수 있으니까 과감한 합체 가능
- **수집 요소**: 전서 등록률 = 유닛 도감 완성도
- **경제 연결**: 재구매 비용이 영혼력(전서 포인트) 지출 → 경영과 연결
- **단계적 개방**: 처음엔 기초 유닛만, 진행할수록 전서가 넓어짐

### 2. 합체 (메가텐식 — 인(印) 기반)

기존 유닛 2체를 소재로 사용하여 **완전히 새로운 유닛**을 생성.
A를 강화하는 것이 아니라, A와 B가 소멸하고 C가 탄생하는 구조.

> **결과 유닛(C)은 인(印) 테이블로 정해지지만, C의 성질은 소재(A, B)의 육성 이력이 결정한다.**

#### 합체의 구조

```
유닛 A (인: 백양, 레벨 15, 육성 이력: 전투형)
  +
유닛 B (인: 거해, 레벨 12, 육성 이력: 조교형)
  ↓
[인(印) 테이블 참조] 백양 × 거해 = 천갈 (15×15 테이블)
[레벨 결정] (15 + 12) / 2 ≈ 14 → 천갈 인 유닛 중 레벨 14 근처
[트레잇 계승] 3경로 판정 (합성 발동 → 직접 계승 → 잠재력 계승)
  ↓
유닛 C (인: 천갈, 레벨 14, A와 B의 트레잇을 3경로로 계승)
```

#### 합체 테이블: 인(印) 15×15

합체 테이블의 키는 **인(印)**이다 (대분류/원소가 아님).
12정인 + 3특수인(염/수은/유황) = 15종이 동등하게 참여하는 15×15 테이블.

- 인A × 인B = 인C로 결과 인이 결정
- 소재 레벨 평균으로 해당 인 내 구체 유닛이 결정
- 특수인도 정인과 동등하게 테이블에 참여

> 합체 테이블 구체 내용 (어떤 인×인이 어떤 인이 되는가)은 미확정.
> 인(印) 체계 상세는 → system_unit_sigil.md 참조

#### 핵심: 같은 공식이라도 결과가 다르다

```
[케이스 1]
  화염 임프(전투형 육성) + 물요정(전투형 육성) = 결과 유닛 C (전투 트레잇 강하게 직접 계승)

[케이스 2]
  화염 임프(조교형 육성) + 물요정(교류형 육성) = 결과 유닛 C (성격/성인 트레잇 직접 계승)

→ 같은 화염 임프 + 물요정 조합인데, 소재를 어떻게 키웠느냐에 따라 C의 트레잇이 달라짐
→ "어떤 트레잇을 키워서 합체에 넣을까"가 판단 포인트
```

#### 합체 규칙

- **인(印)×인(印) 15×15 테이블** 기반으로 결과 인 결정 (메가텐식)
- 소재 레벨 평균으로 결과 레벨 범위 결정
- 소재의 **트레잇**이 3경로 계승 판정의 입력값
- 트레잇 계승: **합성 발동 → 직접 계승 → 잠재력 계승** (3경로)
- 합체 시 소재 유닛은 소멸 (전서에 등록되어 있으면 재구매 가능)
- 합체 결과 유닛도 전서에 등록됨
- 특수 합체: 지정된 유닛 조합에서만 나오는 특수 결과 (별도 테이블)
- **특수인(염/수은/유황)이 소재일 때**: 계승 3경로에서 특수 효과 발휘
  - 염: 직접 계승 슬롯 +1
  - 수은: 합성 발동 조건 완화
  - 유황: 잠재력 인하량 2배

### 3. 이벤트 영입

특수 조건에서만 만날 수 있는 유닛.

- **던전 조우**: 특정 층/조건에서 유닛으로 조우 → 선택지에 따라 영입
- **마일스톤 보상**: 특정 달성 시 자동 영입
- **노드 분기**: 특정 노드 선택에 따라 영입 가능/불가 분기
- **관계 연쇄**: 기존 유닛의 호감도/이벤트가 새 유닛 영입 조건이 되는 경우
- 이벤트 영입 유닛도 전서에 등록 → 이후 재구매 가능

## 유닛 스탯 구조

### 스탯 블록

```
[기본 정보]
- 유닛 ID, 이름
- 레벨, 경험치

[3축 분류] ← v0.9 추가
- 대분류 (세계관적 존재 유형): 요괴/정령/인조/야수/환상 (확정 5종, 방문자는 태그로 분리)
  → 조교 경향 범위, 납품 소폭 보정, 도시 배치 시설 유형 적성, 로어, 서식지
- 인(印) (마력 응축 패턴): 12정인 + 3특수인 = 15종
  → 합체 테이블의 키 (A×B=C), 조교 반응 가중치, 트레잇 성장 방향
- 원소 속성
  → 방어 프로필 (6슬롯: 물리/열/위/동/광/식), 태그 간섭, 도시 배치 적합도
- 3축은 독립적으로 작동하며 서로 간섭하지 않음
  → 상세: system_unit_sigil.md 참조

[전투 — 장비 의존]
- HP: 대분류 기본값 + 장비 보정
- 공격/방어/속도: 주로 장비에 의해 결정
- 원소 방어 프로필 (6슬롯: 물리/열/위/동/광/식) — 장비로 변경 가능
- 유닛 자체의 전투 수치는 기초 방어 프로필 정도만 보유
- → 유닛의 전투력은 "누구인가"보다 "뭘 장착했는가"에 좌우

[트레잇] ← 유닛의 정체성을 결정하는 핵심
- 스킬 트레잇: 전투 중 사용 가능한 능력 (육성/계승으로 획득)
- 성격 트레잇: 이벤트 반응, 육성 방향, 관계에 영향
- 성인 트레잇: ERA식 조교 특성 계열
- 신체 트레잇: 전투/탐사 보정
- 트레잇에 장단점 구분 없음 — 맥락에 따라 쓸모가 달라짐
- 트레잇은 육성으로 추가/성장하며, 합체로 다른 유닛의 트레잇을 조합 가능

[관계]
- 호감도 (수치)
- 이벤트 플래그 (어떤 이벤트를 겪었는가)
- 노드 선택 기억 (해당 유닛과 관련된 노드에서의 선택 이력)

[합체]
- 종족/타입 (합체 테이블 참조용)
- 레벨 (합체 결과 결정에 사용)
- 트레잇 목록 (계승 판정의 입력값)

[로어]
- 해금 가능 파편 목록 (호감도 단계별)
- 출신/배경 태그 (이벤트 트리거 조건으로 활용)

[육성 이력] ← ERA 참고
- 성장 경로 태그 (어떤 방식으로 육성했는가의 누적 기록)
- 트레잇 성장 이력 (어떤 트레잇이 얼마나 성장했는가)
- 경험 이력 (동행 던전, 참여 이벤트, 겪은 노드 선택 등)
```

## 트레잇 시스템

### 핵심: 유닛의 정체성은 수치가 아니라 트레잇 조합이다

전투 수치는 장비가 담당하고, 유닛 자체는 **트레잇의 집합**으로 정의된다.
같은 종족이라도 어떤 트레잇을 가졌느냐에 따라 전혀 다른 유닛이 된다.

### 트레잇 카테고리

| 카테고리 | 예시 | 영향 범위 |
|---------|------|----------|
| 스킬 | 화염 마법, 연속 공격, 회복술 | 전투 중 행동 |
| 성격 | 심약, 호기심, 냉정, 순종적 | 이벤트 반응, 육성 분기, 관계 톤 |
| 성인 | 공포각인, 굴복, 신뢰, 수치심없음, 유아퇴행 등 (ERA식) | 관계 이벤트, 납품 가치, 육성 분기 |
| 신체 | 약체질, 강건, 민첩 등 | 전투/탐사 보정 |

#### 트레잇에 "장단점" 구분은 없다

> **트레잇은 트레잇일 뿐. 맥락에 따라 쓸모가 달라진다.**

- 심약함은 전투에서는 불리하지만, 특정 육성 방향의 전제 조건이 될 수 있음
- 반항적은 조교에서는 어렵지만, 전투에서는 공격 보정이 될 수 있음
- **합체의 핵심 목적: 원하는 트레잇 조합(빌드)을 만드는 것**
  - A에 심약함이 있고 B에 수치심없음이 있으면 합체해서 C에 둘 다 실리게
  - 단일 유닛 육성만으로는 만들 수 없는 트레잇 조합을 합체로 구현
- 어떤 트레잇이 "좋은" 것인지는 플레이어의 목적에 따라 달라짐
  - 전투용 빌드와 납품용 빌드와 관계용 빌드가 원하는 트레잇이 다름

### 트레잇 획득 시스템 (v1.0 — 경험치 풀 + 임계점 + 확률)

> 상세: system_unit_sigil.md 참조

트레잇은 고정이 아니라 **육성에 따라 획득/강화**된다.
획득은 "경험치 풀 → 임계점 → 확률 판정 → 랜덤 해금" 구조를 따른다.

#### 경험치 풀 구조

하나의 경험치 풀이 **여러 트레잇 후보**를 담당한다.
행동으로 경험치가 쌓이면, 임계점 도달 후 확률 판정으로 풀 내 트레잇 중 하나가 해금된다.

```
[경험치 풀]    → [트레잇 후보군] (풀 내 랜덤)
전투 경험치    → 검술/창술/도술/화염마법/빙결마법/연속공격/...
신체 경험치    → 강건/민첩/야성/감각/...
성격 경험치    → 심약/호기심/냉정/반항적/순종적/...
성인 경험치    → 굴복/순종/수치심/신뢰/공포각인/... (상위는 도구로 접근 범위 결정)
```

#### 행동 → 경험치: 주+부 동시 획득

하나의 행동이 **주 경험치 + 부 경험치**를 동시에 올린다.

| 행동 | 주 경험치 | 부 경험치 |
|------|----------|----------|
| 전투 동행 | 전투 | 신체 |
| 훈련 | 전투 | — |
| 탐사 동행 | 신체 | 전투 |
| 교류 (대화/선물) | 성격 | — |
| 조교/교육 | 성인 | 성격 |
| 도시 배치 | (별도 검토) | — |

- "전투만 시켜도 신체가 조금씩 오른다", "조교하면 성격도 조금 변한다"

#### 해금 판정: 임계점 + 확률

```
경험치 < 임계점: 판정 자체가 발생하지 않음
경험치 ≥ 임계점: 매 관련 행동 시 확률 판정 발생
경험치가 더 쌓일수록: 판정 확률 상승
해금 시: 풀 내 후보 중 하나가 랜덤으로 열림
```

- 같은 유닛을 같은 방식으로 키워도 다른 트레잇이 열림 → 다양한 조합의 핵심 장치
- 한번 해금된 트레잇은 풀에서 제거 → 다음 해금 시 나머지 후보 중 판정

#### 해금 가중치

**인(印)**: 특정 풀의 경험치 획득 배율 + 풀 내 특정 트레잇 가중치
- 백양(하소): 전투 풀 속도 UP + 풀 내 공격계 트레잇 가중치 UP

**도구** (crafting 시스템 연결): 상위 트레잇 접근 허가 + 특정 트레잇 가중치
- Tier 1 트레잇: 도구 없이 해금 가능
- Tier 2+ 트레잇: 해당 도구가 있어야 후보 풀에 포함됨 (도구 = 잠금 해제 키)
- 예: 기초 조교도구 → ERA 기초 성인 트레잇 후보 개방
- 예: 상위 구속구 → 공포각인 계열 후보 개방

**동행 유닛 가중치: 기각** — 시스템 간 연쇄가 과도해지고, 하나의 트레잇으로 수렴하는 구조는 다양한 조합을 유도해야 하는 게임 방향과 맞지 않음.

#### 미해금 경험치와 합체 잠재력의 연결

경험치만 쌓이고 트레잇이 안 열린 상태에서 합체 소재로 쓰면:
- 해당 풀의 **임계점 인하**로 결과물에 잠재력 계승
- 경험치가 높을수록 인하량이 큼
- 결과물이 같은 풀에서 트레잇을 열 때 더 빠르게 판정 기회를 얻음

```
예시:
  소재 A: 전투 경험치 높음, 트레잇 미해금
  → 합체 → 결과물 C의 전투 풀 임계점 인하
  → C가 전투 경험치를 조금만 쌓아도 판정 기회 발생
```

- **육성 = 경험치 풀에 축적 → 임계점 넘기면 랜덤 해금**
- **합성 = 해금된 트레잇이 조합 조건을 충족하면 발현**
- **합체 잠재력 = 미해금 경험치까지 가치가 있다**

### 단일 육성 vs 합체의 차이

- **단일 육성**: 한 유닛 안에서 트레잇을 쌓아감. 한 방향으로는 깊게 갈 수 있지만 폭이 제한적
- **합체**: 서로 다른 방향으로 키운 유닛들의 트레잇을 합쳐서 **한 유닛에서는 불가능한 조합**을 만듦
  - 예: A를 전투 특화로 키우고, B를 성인 특성 특화로 키운 뒤 합체 → C에 양쪽 트레잇이 실림
  - 이것이 합체의 핵심 존재 이유: **트레잇 빌드의 폭을 넓히는 수단**

## 합체 계승 시스템 (v0.9 — 3경로 구조)

> 레퍼런스: 메가텐(스킬 직접 계승) + 우마무스메(인자/잠재력 누적) + 자체 조합 시스템(합성 발동)
> 상세: system_unit_sigil.md 참조

### 핵심: 트레잇 기반 3경로 계승

합체 시 결과 유닛 C의 **인(印)은 15×15 테이블로 결정**되지만,
C가 가지는 **트레잇은 소재 A, B의 트레잇에서 3경로로 계승**된다.

### 3경로 구조

```
소재 A의 트레잇들 + 소재 B의 트레잇들
          ↓ 합체 시 판정
  ┌───────┼───────┐
  ↓       ↓       ↓
합성 발동  직접 계승  잠재력 계승
(A+B→새것) (그대로)  (임계점↓)
```

**경로 1: 합성 발동 (최우선)**
- 소재 A+B의 트레잇이 합성 조건을 충족하면 → 합성 트레잇으로 변환돼서 계승 슬롯에 우선 배치
- **서로 다른 유닛의 트레잇끼리** 합성 가능 → 단일 육성에서 불가능한 조합이 열림
- 합성에 사용된 원본 트레잇은 소멸 (합성 재료로 소비됨)
- 조합 시스템의 "특수 레시피 우선 참조"와 동일한 문법

**경로 2: 직접 계승 (남은 슬롯)**
- 합성에 쓰이지 않은 나머지 트레잇 중, 계승 슬롯 수만큼 선택 (성장도 높은 순)
- 메가텐의 스킬 계승과 동일한 구조
- 결과물에 그대로 넘어감

**경로 3: 잠재력 계승 (나머지 전부)**
- 직접 계승 슬롯에서도 밀려난 트레잇 → **버려지지 않고** 결과물의 해당 트레잇 획득 임계점을 낮춤
- 우마무스메의 인자(因子) 시스템과 동일한 원리
- "지금은 없지만 빨리 배울 수 있다" 상태로 결과물에 잠재

### 판정 흐름

```
1단계: 합성 체크 (최우선)
  A의 트레잇과 B의 트레잇 전체를 스캔
  합성 조건 충족 쌍이 있으면 → 합성 트레잇으로 변환, 계승 슬롯에 우선 배치
  합성에 사용된 원본 트레잇은 소멸
  ↓
2단계: 직접 계승 (남은 슬롯)
  합성에 쓰이지 않은 나머지 트레잇 중
  계승 슬롯 수만큼 선택 (성장도 높은 순)
  → 결과물에 그대로 계승
  ↓
3단계: 잠재력 계승 (나머지 전부)
  직접 계승에서도 밀려난 트레잇
  → 결과물의 해당 트레잇 임계점 인하로 전환
  → 소재의 성장도에 비례하여 인하량 결정
```

이 판정 흐름은 조합 시스템의 "태그 합산 → 특수 레시피 우선 → 자동 생성 → 미지" 판정과 동일한 문법 패턴을 따른다.

### 합체 체인의 가치: 잠재력 누적

합체를 반복할수록 잠재력이 누적된다.
- 단발 합체: 즉각적 결과, 단순한 계승
- 다세대 합체: 시간 투자 필요, 하지만 단일 육성/단발 합체로 불가능한 트레잇 조합 도달 가능
- **세대를 쌓는다 = 잠재력을 축적한다 = 다음 세대의 가능성이 넓어진다**

### 트레잇 합성 (조합 발현)

> **특정 트레잇 조합이 한 유닛에 모이면, 새로운 상위 트레잇이 발현된다.**
> 아이템 조합에서 태그 조합이 결과물을 결정하는 것과 동일한 구조.

```json
{
  "synthesis_id": "TSYN_001",
  "name": "트리거 해피",
  "required_traits": ["낮은 집중력", "사격", "공격적", "연속타격"],
  "category": "스킬",
  "effect": "전투 시 랜덤 대상에게 다수 사격, 명중률 감소",
  "lore": "누구를 맞추는지는 본인도 모른다. 다만 방아쇠에서 손을 떼지 못할 뿐."
}
```

```json
{
  "synthesis_id": "TSYN_002",
  "name": "유아퇴행",
  "required_traits": ["심약", "오줌싸개", "수치심없음"],
  "category": "성인",
  "effect": "성인 특성 상호작용 변화, 특수 이벤트 해금, 도시 유닛 요청 대응 가능",
  "lore": "어른의 몸에 아이의 마음이 깃들었다. 돌봄이 필요하다."
}
```

#### 트레잇 합성의 특성

- **합체의 진짜 목적**: 단순히 트레잇을 모으는 게 아니라, 합성 트레잇을 노리는 것
- **레시피 발견의 재미**: 어떤 조합이 어떤 상위 트레잇을 만드는지 플레이어가 발견
- **육성 계획의 깊이**: "트리거 해피를 만들려면 A를 이렇게 키우고 B를 저렇게 키워서 합체"
- **합성 트레잇도 계승 가능**: 합성으로 만들어진 트레잇을 가진 유닛을 다시 합체 소재로 쓰면 계승 가능
- **아이템 시스템과 문법 통일**: 재료 태그 조합 → 아이템 / 트레잇 조합 → 상위 트레잇
- **합성 레시피에도 로어 텍스트 부착** → 파편 로어와 연결

#### 트레잇 합성의 발동 타이밍

- **합체 시 (경로 1)**: 소재 A+B의 트레잇이 합성 조건 충족 → 가장 일반적. **서로 다른 유닛의 트레잇끼리** 합성 가능
- **육성 중**: 한 유닛 안에서 트레잇을 쌓다가 합성 조건 충족 → 발견의 재미
  - 예: 훈련으로 사격 획득 → 조교로 공격적 획득 → 추가 훈련으로 연속타격 획득 → 이미 가지고 있던 낮은 집중력과 합쳐져서 트리거 해피 발현
  - 단일 육성으로도 합성 트레잇을 만들 수 있지만, 필요한 트레잇을 한 유닛에서 전부 키우기 어려움 → 합체가 효율적 경로

### 3경로 계승 예시

```
[합성 발동 — 경로 1]
소재 A가 "심약" 보유 + 소재 B가 "수치심없음" 보유
→ 합성 조건 충족: 심약 + 수치심없음 → "유아퇴행" 합성 발동
→ 결과물 C에 "유아퇴행"이 계승 (심약, 수치심없음은 소멸)

[직접 계승 — 경로 2]
합성에 쓰이지 않은 트레잇 중 성장도 순으로 슬롯만큼 선택
→ 화염 마법 Lv3 → 결과물에 그대로 넘어감

[잠재력 계승 — 경로 3]
직접 계승 슬롯에서도 밀려난 트레잇
→ 소재 A의 "강건" → 결과물 C의 "강건" 획득 임계점이 낮아짐
→ C가 훈련으로 강건을 배울 때 필요한 경험치가 감소

[크로스 빌드 — 소재 설계]
목표: 천갈 인 + "전투광" + "신뢰" 트레잇을 가진 유닛 C
역산: 백양(훈련 반응 높음) × 거해(교류 반응 높음) = 천갈
  소재 A(백양) → 전투광 트레잇 육성 (인에 맞는 육성 = 효율적)
  소재 B(거해) → 신뢰 트레잇 육성 (인에 맞는 육성 = 효율적)
  합체 → C(천갈)에 전투광 + 신뢰 계승
```

### 재육성 보너스

합체로 태어난 유닛 C를 다시 육성할 때:

- **잠재력 계승**을 받은 트레잇은 획득 임계점이 낮아져 빠르게 배울 수 있음
- 직접 계승받은 트레잇은 이미 보유 → 즉시 활용 가능
- 인(印)에 맞는 방향으로 육성하면 추가 효율 → "인이 경사면이지 벽이 아니다"
- → "합체에서 좋은 기반을 만들고, 재육성으로 완성한다"는 2단계 성장 루프

## 육성 시스템

### 핵심: 육성 과정 자체가 분기 (ERA식)

같은 유닛이라도 **어떤 행동을 시켰느냐, 어떤 도구를 썼느냐, 어떤 이벤트를 겪었느냐**에 따라
획득하는 특성이 달라지고, 그 특성이 유닛의 성질을 바꾼다.

### 육성 방법

| 방법 | 스태미나 | 효과 | 비고 |
|------|---------|------|------|
| 탐사 동행 | 탐사에 포함 | 전투 경험, 동행 이력 누적, 호감도 변동 | 부상 리스크 |
| 훈련 | 중간 | 스탯 성장, 전투 특성 획득 | 육성 도구 영향 |
| 조교/교육 | 중간~높음 | 성격 특성, 성인 특성 분기 | ERA식 핵심 육성 |
| 교류 | 낮음 | 호감도 상승, 성격 변화, 관계 이벤트 | 관계 시스템의 핵심 |
| 도시 배치 | 없음 (도시 시설 배치) | 시설 가동, 재료 수급, 경험 누적 | 도시 시설을 운영하는 유닛 |

### 인(印)과 육성 반응 (v0.9 추가)

인(印)이 육성 **초기 경향의 범위**를 정한다 (포켓몬 성격치 방식).
같은 인이라도 개체마다 구체적 성격은 다르지만, 인이 범위를 설정하고 개체가 그 안에서 랜덤.

**인이 결정하는 것:**
- **경험치 풀별 획득 배율**: 인의 공정 원리가 풀별 속도를 결정
  - 백양(하소): 전투 1.3x, 신체 1.1x, 성격 0.8x, 성인 0.8x
  - 거해(용해): 전투 0.7x, 신체 0.9x, 성격 1.2x, 성인 1.3x
  - (수치는 밸런싱 시 확정)
- **풀 내 트레잇 해금 가중치**: 같은 풀 안에서도 인에 따라 특정 트레잇이 열리기 쉬움
- 트레잇 합성 친화: 특정 합성 트레잇이 발현되기 쉬운 인이 있음

**인이 결정하지 않는 것:**
- 구체적 초기 성격 (개체별 랜덤)
- 최종 트레잇 구성 (육성으로 얼마든지 변경 가능)
- 호감도 상한 (모든 유닛이 MAX 가능)

**핵심: 인은 경사면이지 벽이 아니다**
- 인에 맞는 육성 → 경험치 빠르게 쌓이고, 원하는 트레잇이 해금될 확률 높음 (효율 추구)
- 인을 거슬러 육성 → 느리지만 독특한 트레잇 조합, 오래 키우면서 정이 듦 (애착 육성)

> 인별 경험치 풀 배율, 트레잇 해금 가중치 구체 수치는 미확정.
> 인(印) × 조교 시스템 상세는 → system_unit_sigil.md 참조

### 육성 방향이 영향을 주는 범위

```
육성 이력 →
  → 트레잇 획득/성장/변화 방향
  → 합체 시 계승 트레잇 풀 결정
  → 탐사 시 반응/행동 패턴
  → 이벤트 발생 조건 및 선택지 변화
  → 납품/거래 시 영혼력 가치 산정 (좋은 트레잇 = 높은 영혼력)
  → 관계 이벤트의 톤과 분기
```

### 육성 도구의 역할

- 육성 도구(훈련 도구, 조교 도구)에 따라 육성 방향이 달라짐
- 특정 도구가 특정 특성 계열을 개방하거나 확률을 올림
- 도구도 조합으로 제작 → 조합 시스템과 연결
- 상위 도구 = 상위 특성 접근 가능 (단계적 개방)

## 관계 시스템

### 호감도

- 수치 기반 (0 ~ MAX)
- 호감도 단계별 변화:
  - 낮음: 기본 대사, 최소한의 반응
  - 중간: 개인 이벤트 해금, 탐사 시너지 발생
  - 높음: 고유 스킬 해금, 특수 대사, 조합 보너스
  - MAX: 연애 루트 진입 가능, 전용 엔딩 조건

### 호감도 변동 요인

```
상승:
  - 탐사 동행
  - 교류 (대화, 선물 등)
  - 유닛이 좋아하는 방향의 육성
  - 유닛 관련 이벤트에서 유닛 편을 드는 선택
  - 유닛 부상 시 치료 아이템 사용

하락:
  - 유닛이 싫어하는 방향의 육성
  - 유닛 관련 이벤트에서 유닛에 불리한 선택
  - 같은 파티 유닛 간 상성 충돌 (미결정)
  - 장기간 방치 (미결정)
```

### 연애 루트

> **"안 해도 되지만 있으면 하게 되는"**

- 호감도 MAX 도달 + 특정 이벤트 플래그 충족 시 진입 가능
- 연애 루트 진입 시 전용 이벤트 체인 개시
- 연애 중인 유닛을 합체 소재로 쓰면? → 이머전트 이벤트 (감정적 고민)
- 연애 중인 유닛을 납품하면? → 이머전트 이벤트 (감정적 고민)
- 연애 루트 완료 = 관계 엔딩 (2-X) 해금 조건
- 복수 연애 가능 여부 (미결정)

### 유닛 간 관계

유닛끼리도 관계가 존재할 수 있다.

- 같이 탐사 다니면 유닛 간 친밀도 변동
- 특정 유닛 조합이 시너지/상성 충돌을 일으킴
- 유닛 간 관계가 이벤트 트리거 조건에 영향
- **유닛 A와 B 중 하나를 합체 소재로 써야 할 때, 남은 유닛의 반응** → 이머전트 이벤트

---

## 유닛과 다른 시스템의 간섭

```
[유닛 → 탐사]
  - 파티 편성: 유닛 조합이 전투력/시너지를 결정
  - 동행 시 호감도 변동 + 육성 이력 누적
  - 유닛 부상 → 파티 약화

[유닛 → 조합]
  - 합체 소재: 유닛이 소멸하여 새 유닛 생성
  - 육성 이력이 합체 결과에 영향
  - 높은 호감도 유닛을 소재로 쓰면 감정적 비용 발생

[유닛 → 경영]
  - 납품 대상: 잘 키운 유닛은 높은 영혼력 획득
  - 유지비 발생: 유닛 수/등급에 비례
  - 도시 배치: 대분류가 배치 가능 시설 유형 범위를 결정, 트레잇이 보너스 크기를 결정
    → "이 인조를 공방에 배치하고, 스킬 트레잇으로 제작 효율을 올린다"
  - 도시 유닛 요청: "이런 특성의 유닛 원함"

[유닛 → 내러티브]
  - 로어 캐리어: 호감도 단계별 파편 로어 해금
  - 이벤트 트리거: 유닛의 출신/배경 태그가 이벤트 조건
  - 노드 분기: 특정 유닛 보유/관계가 노드 출현 조건
  - 관계 엔딩: 연애 루트 완료 = 2-X 엔딩

[유닛 ← 달력]
  - 유닛별 기념일/이벤트 (생일 등)
  - 시즌에 따른 유닛 행동/대사 변화
```

## 미결정 사항

### 확정됨 (v0.9)
- [x] 유닛 종족/타입 체계 — 5대 분류 확정 (요괴, 정령, 인조, 야수, 환상) + 방문자는 태그로 분리 → world_setting.md 참조
- [x] **유닛 분류 3축 체계**: 대분류(세계관적 존재유형) + 인(印)(마력 응축 패턴) + 원소 속성. 각 축은 독립적 역할
- [x] **합체 테이블 키**: 인(印) 15×15 테이블 (12정인 + 3특수인) → system_unit_sigil.md
- [x] **트레잇 계승 방식**: 3경로 혼합 (합성 발동 + 직접 계승 + 잠재력 계승)
- [x] **계승 판정 우선순위**: 합성 발동 → 직접 계승 → 잠재력 계승 (조합 시스템 판정 문법과 동일 패턴)
- [x] **잠재력 누적**: 합체 체인(세대)을 거듭할수록 누적 — 합체 반복의 장기 투자 동기
- [x] **인의 조교 관여**: 인이 조교 반응 가중치, 트레잇 성장 방향 가중치, 합성 친화를 결정 (포켓몬 성격치 방식)
- [x] **인은 경사면이지 벽이 아니다**: 인에 맞는 육성 = 효율, 인을 거스르는 육성 = 느리지만 가능
- [x] **특수인(염/수은/유황)**: 15×15 테이블 정상 참여, 계승 3경로에서 특수 효과
- [x] **트레잇 획득 메커니즘**: 경험치 풀 + 임계점 + 확률 판정. 1풀 = 다중 후보, 랜덤 해금
- [x] **경험치 구조**: 주+부 동시 획득 (전투 시 전투+신체, 조교 시 성인+성격)
- [x] **도구 역할**: 상위 트레잇 접근 허가 = 잠금 해제 키 (crafting 시스템 연결)
- [x] **동행 유닛 가중치: 기각** (시스템 연쇄 과도, 다양성 저해)
- [x] **미해금 경험치의 가치**: 합체 시 잠재력 계승으로 전환 (경험치까지 가치 있음)
- [x] **대분류의 시스템적 역할**: 도시 배치 시설 유형 적성 담당. 대분류→시설 유형 범위, 트레잇→보너스 크기. 3축 각각 고유 영역(대분류=경영, 인=합체+육성, 원소=전투)
