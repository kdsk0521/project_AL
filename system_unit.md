## 개요

유닛은 이 게임에서 가장 복합적인 존재.
하나의 유닛이 **전투원 + 관계 대상 + 합체 소재 + 로어 캐리어 + 판매 상품**의 역할을 동시에 가진다.

> **"같은 유닛이라도 어떻게 키웠느냐에 따라 다른 존재가 된다."** (ERA 참고)

---

## 설계 원칙

- 유닛은 스탯 덩어리가 아니라 **감정이 붙는 대상**
- 육성 과정이 곧 분기 → 같은 종류의 유닛이라도 플레이어마다 다른 결과
- 합체 소재로 쓸 때, 팔 때 **고민이 생기는** 구조 (ERA식 감정 비용)
- 관계(호감도, 연애)는 "안 해도 되지만 있으면 하게 되는" 서브 시스템

## 유닛 획득 경로

유닛을 얻는 방법은 3가지.

### 1. 가게 구매 (유닛 전서)

페르소나 악마전서와 동일한 구조.

- **기초 풀**: 게임 시작 시 구매 가능한 기본 유닛 몇 종
- **해금 확장**: 마일스톤/엔딩 달성 시 새로운 유닛이 전서에 추가
- **전서 등록**: 한번 합체로 만들었거나 이벤트로 영입한 유닛은 전서에 등록 → 이후 재구매 가능
- 재구매 시 **기본 상태로 초기화** (육성 이력/호감도는 리셋)
- 전서 등록률 자체가 수집 요소 (유닛 도감과 연동)

```json
{
  "compendium_entry": "UNIT_FIRE_IMP",
  "name": "화염 임프",
  "base_price": 500,
  "unlock_type": "default",
  "registered": true,
  "first_obtained": "합체"
}
```

#### 전서의 역할

- **합체 부담 경감**: 소재로 써도 다시 살 수 있으니까 과감한 합체 가능
- **수집 요소**: 전서 등록률 = 유닛 도감 완성도
- **경제 연결**: 재구매 비용이 자금 지출 → 경영과 연결
- **단계적 개방**: 처음엔 기초 유닛만, 진행할수록 전서가 넓어짐

### 2. 합체 (메가텐식 — 별도 문서)

기존 유닛 2체를 소재로 사용하여 **완전히 새로운 유닛**을 생성.
A를 강화하는 것이 아니라, A와 B가 소멸하고 C가 탄생하는 구조.

> **결과 유닛(C)은 테이블로 정해지지만, C의 성질은 소재(A, B)의 육성 이력이 결정한다.**

#### 합체의 구조

```
유닛 A (종족 X, 레벨 15, 육성 이력: 전투형)
  +
유닛 B (종족 Y, 레벨 12, 육성 이력: 조교형)
  ↓
[종족 테이블 참조] X × Y = 종족 Z
[레벨 결정] (15 + 12) / 2 = 레벨 13~14의 종족 Z 유닛
[특성 계승] A의 전투 특성 + B의 조교 특성 중 일부가 C에 계승
  ↓
유닛 C (종족 Z, 레벨 14, A와 B의 특성을 일부 계승)
```

#### 핵심: 같은 공식이라도 결과가 다르다

```
[케이스 1]
  화염 임프(전투형 육성) + 물요정(전투형 육성) = 결과 유닛 C (전투 특성 강하게 계승)

[케이스 2]
  화염 임프(조교형 육성) + 물요정(교류형 육성) = 결과 유닛 C (성격/성인 특성 계승)

→ 같은 화염 임프 + 물요정 조합인데, 소재를 어떻게 키웠느냐에 따라 C의 성질이 달라짐
→ "어떤 A를 만들어서 합체에 넣을까"가 판단 포인트
```

#### 합체 규칙

- 종족×종족 테이블 기반으로 결과 종족 결정 (메가텐식)
- 소재 레벨 평균으로 결과 레벨 범위 결정
- 소재의 **육성 이력**이 결과물의 특성 계승 방향에 영향
- 스킬 계승 규칙 (미결정 — 랜덤? 선택? 육성 이력 기반?)
- 합체 시 소재 유닛은 소멸 (전서에 등록되어 있으면 재구매 가능)
- 합체 결과 유닛도 전서에 등록됨
- 특수 합체: 지정된 유닛 조합에서만 나오는 특수 결과 (별도 테이블)

### 3. 이벤트 영입

특수 조건에서만 만날 수 있는 유닛.

- **던전 조우**: 특정 층/조건에서 유닛으로 조우 → 선택지에 따라 영입
- **마일스톤 보상**: 특정 달성 시 자동 영입
- **노드 분기**: 특정 노드 선택에 따라 영입 가능/불가 분기
- **관계 연쇄**: 기존 유닛의 호감도/이벤트가 새 유닛 영입 조건이 되는 경우
- 이벤트 영입 유닛도 전서에 등록 → 이후 재구매 가능

## 유닛 스탯 구조

### 스탯 블록

```
[기본 정보]
- 유닛 ID, 이름, 종족/타입
- 레벨, 경험치

[전투 — 장비 의존]
- HP: 종족 기본값 + 장비 보정
- 공격/방어/속도: 주로 장비에 의해 결정
- 속성/타입 (상성 시스템과 연결 — 미결정)
- 유닛 자체의 전투 수치는 기초 방어 카테고리 정도만 보유
- → 유닛의 전투력은 "누구인가"보다 "뭘 장착했는가"에 좌우

[트레잇] ← 유닛의 정체성을 결정하는 핵심
- 스킬 트레잇: 전투 중 사용 가능한 능력 (육성/계승으로 획득)
- 성격 트레잇: 이벤트 반응, 육성 방향, 관계에 영향
- 성인 트레잇: ERA식 조교 특성 계열
- 신체 트레잇: 전투/탐사 보정
- 트레잇에 장단점 구분 없음 — 맥락에 따라 쓸모가 달라짐
- 트레잇은 육성으로 추가/성장하며, 합체로 다른 유닛의 트레잇을 조합 가능

[관계]
- 호감도 (수치)
- 이벤트 플래그 (어떤 이벤트를 겪었는가)
- 노드 선택 기억 (해당 유닛과 관련된 노드에서의 선택 이력)

[합체]
- 종족/타입 (합체 테이블 참조용)
- 레벨 (합체 결과 결정에 사용)
- 트레잇 목록 (계승 판정의 입력값)

[로어]
- 해금 가능 파편 목록 (호감도 단계별)
- 출신/배경 태그 (이벤트 트리거 조건으로 활용)

[육성 이력] ← ERA 참고
- 성장 경로 태그 (어떤 방식으로 육성했는가의 누적 기록)
- 트레잇 성장 이력 (어떤 트레잇이 얼마나 성장했는가)
- 경험 이력 (동행 던전, 참여 이벤트, 겪은 노드 선택 등)
```

## 트레잇 시스템

### 핵심: 유닛의 정체성은 수치가 아니라 트레잇 조합이다

전투 수치는 장비가 담당하고, 유닛 자체는 **트레잇의 집합**으로 정의된다.
같은 종족이라도 어떤 트레잇을 가졌느냐에 따라 전혀 다른 유닛이 된다.

### 트레잇 카테고리

| 카테고리 | 예시 | 영향 범위 |
|---------|------|----------|
| 스킬 | 화염 마법, 연속 공격, 회복술 | 전투 중 행동 |
| 성격 | 심약, 호기심, 냉정, 순종적 | 이벤트 반응, 육성 분기, 관계 톤 |
| 성인 | 공포각인, 굴복, 신뢰, 수치심없음, 유아퇴행 등 (ERA식) | 관계 이벤트, 판매 가치, 육성 분기 |
| 신체 | 약체질, 강건, 민첩 등 | 전투/탐사 보정 |

#### 트레잇에 "장단점" 구분은 없다

> **트레잇은 트레잇일 뿐. 맥락에 따라 쓸모가 달라진다.**

- 심약함은 전투에서는 불리하지만, 특정 육성 방향의 전제 조건이 될 수 있음
- 반항적은 조교에서는 어렵지만, 전투에서는 공격 보정이 될 수 있음
- **합체의 핵심 목적: 원하는 트레잇 조합(빌드)을 만드는 것**
  - A에 심약함이 있고 B에 수치심없음이 있으면 합체해서 C에 둘 다 실리게
  - 단일 유닛 육성만으로는 만들 수 없는 트레잇 조합을 합체로 구현
- 어떤 트레잇이 "좋은" 것인지는 플레이어의 목적에 따라 달라짐
  - 전투용 빌드와 판매용 빌드와 관계용 빌드가 원하는 트레잇이 다름

### 트레잇 성장

- 트레잇은 고정이 아니라 **육성에 따라 획득/강화**된다
- 스킬 트레잇: 사용/훈련할수록 강화 (화염 마법 Lv1 → Lv3)
- 성격/성인/신체 트레잇: 육성 방향에 따라 새로운 트레잇이 **개별적으로 획득**됨
  - 트레잇 간에 선형 진화 관계는 없음 (심약이 진화해서 유아퇴행이 되는 게 아님)
  - 각 트레잇은 독립적으로 획득되는 **재료**
  - 특정 트레잇 조합이 모이면 합성 트레잇이 **자동 발현** (트레잇 합성)
  - 예: 심약(육성 획득) + 오줌싸개(육성 획득) + 수치심없음(육성 획득) → 유아퇴행(합성 발현)
- 경험치는 유닛 전체 레벨뿐 아니라 **개별 트레잇 성장**에도 관여
- **육성 = 개별 트레잇을 모으는 과정, 합성 = 모은 트레잇이 조합 조건을 충족하면 발현**

### 단일 육성 vs 합체의 차이

- **단일 육성**: 한 유닛 안에서 트레잇을 쌓아감. 한 방향으로는 깊게 갈 수 있지만 폭이 제한적
- **합체**: 서로 다른 방향으로 키운 유닛들의 트레잇을 합쳐서 **한 유닛에서는 불가능한 조합**을 만듦
  - 예: A를 전투 특화로 키우고, B를 성인 특성 특화로 키운 뒤 합체 → C에 양쪽 트레잇이 실림
  - 이것이 합체의 핵심 존재 이유: **트레잇 빌드의 폭을 넓히는 수단**

## 합체 계승 시스템

### 핵심: 트레잇 기반 계승

합체 시 결과 유닛 C의 **종족은 테이블로 결정**되지만,
C가 가지는 **트레잇은 소재 A, B의 트레잇에서 계승**된다.

### 계승 판정 (초안)

```
[소재 A의 트레잇] + [소재 B의 트레잇]
       ↓
[1단계] 트레잇 풀 생성 — A와 B의 모든 트레잇을 후보로 모음
       ↓
[2단계] 성장도 기준 필터 — 일정 수준 이상 성장한 트레잇만 계승 후보
       (우마무스메 인자처럼 "충분히 키운 것만 넘어간다")
       ↓
[3단계] 트레잇 합산/상쇄 판정
       - 같은 트레잇이 양쪽에 있으면: 강화 계승 (Lv 합산/보정)
       - 상반되는 트레잇이 있으면: 상쇄 (둘 다 소멸)
       - 다른 카테고리의 트레잇: 독립적으로 계승 (빌드 조합의 핵심)
       ↓
[4단계] 계승 슬롯 제한 — C가 가질 수 있는 트레잇 수에 상한
       (종족별로 슬롯 수가 다를 수 있음)
       ↓
[5단계] 트레잇 합성 판정 — 특정 트레잇 조합이 C에 모이면 상위 트레잇 발현
       (아이템 조합의 태그 매칭과 동일한 구조)
       ↓
[6단계] 결과 유닛 C 확정 — 종족(테이블) + 트레잇(계승+합성) + 기본 스탯(종족 기준)
```

### 트레잇 합성 (조합 발현)

> **특정 트레잇 조합이 한 유닛에 모이면, 새로운 상위 트레잇이 발현된다.**
> 아이템 조합에서 태그 조합이 결과물을 결정하는 것과 동일한 구조.

```json
{
  "synthesis_id": "TSYN_001",
  "name": "트리거 해피",
  "required_traits": ["낮은 집중력", "사격", "공격적", "연속타격"],
  "category": "스킬",
  "effect": "전투 시 랜덤 대상에게 다수 사격, 명중률 감소",
  "lore": "누구를 맞추는지는 본인도 모른다. 다만 방아쇠에서 손을 떼지 못할 뿐."
}
```

```json
{
  "synthesis_id": "TSYN_002",
  "name": "유아퇴행",
  "required_traits": ["심약", "오줌싸개", "수치심없음"],
  "category": "성인",
  "effect": "성인 특성 상호작용 변화, 특수 이벤트 해금, 관련 손님 주문 대응 가능",
  "lore": "어른의 몸에 아이의 마음이 깃들었다. 돌봄이 필요하다."
}
```

#### 트레잇 합성의 특성

- **합체의 진짜 목적**: 단순히 트레잇을 모으는 게 아니라, 합성 트레잇을 노리는 것
- **레시피 발견의 재미**: 어떤 조합이 어떤 상위 트레잇을 만드는지 플레이어가 발견
- **육성 계획의 깊이**: "트리거 해피를 만들려면 A를 이렇게 키우고 B를 저렇게 키워서 합체"
- **합성 트레잇도 계승 가능**: 합성으로 만들어진 트레잇을 가진 유닛을 다시 합체 소재로 쓰면 계승 가능
- **아이템 시스템과 문법 통일**: 재료 태그 조합 → 아이템 / 트레잇 조합 → 상위 트레잇
- **합성 레시피에도 로어 텍스트 부착** → 파편 로어와 연결

#### 트레잇 합성의 발동 타이밍

- **합체 시**: 소재 A+B의 트레잇이 모여서 합성 조건 충족 → 가장 일반적
- **육성 중**: 한 유닛 안에서 트레잇을 쌓다가 합성 조건 충족 → 발견의 재미
  - 예: 훈련으로 사격 획득 → 조교로 공격적 획득 → 추가 훈련으로 연속타격 획득 → 이미 가지고 있던 낮은 집중력과 합쳐져서 트리거 해피 발현
  - 단일 육성으로도 합성 트레잇을 만들 수 있지만, 필요한 트레잇을 한 유닛에서 전부 키우기 어려움 → 합체가 효율적 경로

### 트레잇 합산/상쇄 예시

```
[합산 — 같은 트레잇이 양쪽에]
A: 화염 적성 Lv2  +  B: 화염 적성 Lv3  →  C: 화염 적성 Lv4 (강화 계승)

[빌드 조합 — 합체의 핵심 용도]
A: 심약, 오줌싸개 (성인 특화 육성)
B: 수치심없음, 노출증 (성인 특화 육성, 다른 방향)
→ C: 심약 + 오줌싸개 + 수치심없음 + 노출증 (단일 육성으로는 불가능한 조합)

[크로스 빌드 — 다른 카테고리 횡단]
A: 화염 마법 Lv3, 연속공격 (전투 특화 육성)
B: 순종적, 신뢰 (관계 특화 육성)
→ C: 전투 스킬 + 관계 트레잇을 동시에 보유 (전투도 되고 관계도 깊은 유닛)

[상쇄 — 상반 트레잇]
A: 반항적  +  B: 순종적  →  둘 다 소멸 (상반 트레잇 상쇄)
```

### 재육성 보너스

합체로 태어난 유닛 C를 다시 육성할 때:

- 계승받은 트레잇 방향으로 육성하면 **성장 보너스** (이미 소질이 있으므로)
- 계승받지 않은 방향으로 키우는 것도 가능하지만 효율은 낮음
- → "합체에서 좋은 기반을 만들고, 재육성으로 완성한다"는 2단계 성장 루프

## 육성 시스템

### 핵심: 육성 과정 자체가 분기 (ERA식)

같은 유닛이라도 **어떤 행동을 시켰느냐, 어떤 도구를 썼느냐, 어떤 이벤트를 겪었느냐**에 따라
획득하는 특성이 달라지고, 그 특성이 유닛의 성질을 바꾼다.

### 육성 방법

| 방법 | 스태미나 | 효과 | 비고 |
|------|---------|------|------|
| 탐사 동행 | 탐사에 포함 | 전투 경험, 동행 이력 누적, 호감도 변동 | 부상 리스크 |
| 훈련 | 중간 | 스탯 성장, 전투 특성 획득 | 육성 도구 영향 |
| 조교/교육 | 중간~높음 | 성격 특성, 성인 특성 분기 | ERA식 핵심 육성 |
| 교류 | 낮음 | 호감도 상승, 성격 변화, 관계 이벤트 | 관계 시스템의 핵심 |
| 실전 배치 | 없음 (가게 배치) | 경영 보너스, 경험 누적 | 가게에서 일하는 유닛 |

### 육성 방향이 영향을 주는 범위

```
육성 이력 →
  → 트레잇 획득/성장/변화 방향
  → 합체 시 계승 트레잇 풀 결정
  → 탐사 시 반응/행동 패턴
  → 이벤트 발생 조건 및 선택지 변화
  → 판매/거래 시 가치 산정 (좋은 트레잇 = 비쌈, 단점 = 감가)
  → 관계 이벤트의 톤과 분기
```

### 육성 도구의 역할

- 육성 도구(훈련 도구, 조교 도구)에 따라 육성 방향이 달라짐
- 특정 도구가 특정 특성 계열을 개방하거나 확률을 올림
- 도구도 조합으로 제작 → 조합 시스템과 연결
- 상위 도구 = 상위 특성 접근 가능 (단계적 개방)

## 관계 시스템

### 호감도

- 수치 기반 (0 ~ MAX)
- 호감도 단계별 변화:
  - 낮음: 기본 대사, 최소한의 반응
  - 중간: 개인 이벤트 해금, 탐사 시너지 발생
  - 높음: 고유 스킬 해금, 특수 대사, 조합 보너스
  - MAX: 연애 루트 진입 가능, 전용 엔딩 조건

### 호감도 변동 요인

```
상승:
  - 탐사 동행
  - 교류 (대화, 선물 등)
  - 유닛이 좋아하는 방향의 육성
  - 유닛 관련 이벤트에서 유닛 편을 드는 선택
  - 유닛 부상 시 치료 아이템 사용

하락:
  - 유닛이 싫어하는 방향의 육성
  - 유닛 관련 이벤트에서 유닛에 불리한 선택
  - 같은 파티 유닛 간 상성 충돌 (미결정)
  - 장기간 방치 (미결정)
```

### 연애 루트

> **"안 해도 되지만 있으면 하게 되는"**

- 호감도 MAX 도달 + 특정 이벤트 플래그 충족 시 진입 가능
- 연애 루트 진입 시 전용 이벤트 체인 개시
- 연애 중인 유닛을 합체 소재로 쓰면? → 이머전트 이벤트 (감정적 고민)
- 연애 중인 유닛을 판매하면? → 이머전트 이벤트 (감정적 고민)
- 연애 루트 완료 = 관계 엔딩 (2-X) 해금 조건
- 복수 연애 가능 여부 (미결정)

### 유닛 간 관계

유닛끼리도 관계가 존재할 수 있다.

- 같이 탐사 다니면 유닛 간 친밀도 변동
- 특정 유닛 조합이 시너지/상성 충돌을 일으킴
- 유닛 간 관계가 이벤트 트리거 조건에 영향
- **유닛 A와 B 중 하나를 합체 소재로 써야 할 때, 남은 유닛의 반응** → 이머전트 이벤트

## 유닛과 다른 시스템의 간섭

```
[유닛 → 탐사]
  - 파티 편성: 유닛 조합이 전투력/시너지를 결정
  - 동행 시 호감도 변동 + 육성 이력 누적
  - 유닛 부상 → 파티 약화

[유닛 → 조합]
  - 합체 소재: 유닛이 소멸하여 새 유닛 생성
  - 육성 이력이 합체 결과에 영향
  - 높은 호감도 유닛을 소재로 쓰면 감정적 비용 발생

[유닛 → 경영]
  - 판매 대상: 잘 키운 유닛은 비싸게 팔림
  - 유지비 발생: 유닛 수/등급에 비례
  - 가게 배치: 유닛이 경영 보너스를 줄 수 있음 (미결정)
  - 손님 주문: "이런 특성의 유닛 원함"

[유닛 → 내러티브]
  - 로어 캐리어: 호감도 단계별 파편 로어 해금
  - 이벤트 트리거: 유닛의 출신/배경 태그가 이벤트 조건
  - 노드 분기: 특정 유닛 보유/관계가 노드 출현 조건
  - 관계 엔딩: 연애 루트 완료 = 2-X 엔딩

[유닛 ← 달력]
  - 유닛별 기념일/이벤트 (생일 등)
  - 시즌에 따른 유닛 행동/대사 변화
```
