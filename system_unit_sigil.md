## 개요

유닛은 3개의 독립된 분류 축을 가진다.
각 축은 담당하는 시스템 영역이 다르며, 서로 간섭하지 않는다.

```
유닛 하나의 분류 정보:
  대분류 (세계관적 존재 유형) → 조교 경향 범위, 납품 소폭 보정, 로어, 서식지
  인(印) (마력 응축 패턴)     → 합체 테이블의 키 (A×B=C)
  원소 속성                   → 방어 프로필 (6슬롯), 태그 간섭, 도시 배치 적합도
```

본 문서는 **인(印) 체계**를 다룬다.

---

## 설계 원칙

- 인(印)의 1차 존재 이유는 **합체 테이블의 키**
- 겉은 황도 12궁(별자리), 속은 연금술 12공정 — 플레이어는 익숙한 별자리로 인식하고, 게임 내에서는 연금술 공정 원리로 작동
- 인은 조교/납품/합체 전 영역에 관여하되, **출발점의 경사면이지 벽이 아니다** — 인을 거슬러 육성하는 것도 가능하지만 비효율적
- 달력(12개월)과 느슨하게 연동 — 1:1 고정이 아니라 매달 활성 인 2~3개가 순환
- 세계관적 근거: 연금술과 점성술은 역사적으로 한 몸. 미궁의 마력에 주기가 있고, 유닛은 태어날 때 그 주기 중 하나를 인(印)으로 가진다

## 인(印) 목록: 12 정인(正印) + 3 특수인(特印) = 15종

### 12 정인 — 황도 12궁 × 연금술 12공정

| # | 인 이름 | 공정 원리 | 아키타입 | 한줄 성질 |
|---|--------|----------|---------|----------|
| 1 | 백양(白羊) | 하소(煅燒) Calcination | 점화자 | 태워서 본질을 드러내는 자. 파괴로 시작을 만듦 |
| 2 | 금우(金牛) | 응고(凝固) Congelation | 형성자 | 흩어진 것에 형태를 부여하는 자. 안정과 축적 |
| 3 | 쌍자(雙子) | 고착(固着) Fixation | 결박자 | 날아가려는 것을 놓지 않는 자. 한번 닿은 것은 자기에게 묶인다 |
| 4 | 거해(巨蟹) | 용해(溶解) Dissolution | 해체자 | 굳은 형태를 녹여 원래로 되돌리는 자. 보호 속의 해방 |
| 5 | 사자(獅子) | 소화(消化) Digestion | 지배자 | 삼켜서 자기 것으로 만드는 자. 열과 의지의 힘 |
| 6 | 처녀(處女) | 증류(蒸溜) Distillation | 정제자 | 불순물을 걸러 순수를 추출하는 자. 분석과 정밀 |
| 7 | 천칭(天秤) | 승화(昇華) Sublimation | 극단자 | 중간이 없는 자. 잠잠하다가 갑자기 폭발하고, 0에서 100으로 |
| 8 | 천갈(天蝎) | 분리(分離) Separation | 해부자 | 뒤섞인 것에서 원하는 것만 떼어내는 자. 집요한 추적 |
| 9 | 인마(人馬) | 연화(軟化) Ceration | 침투자 | 반복으로 스며드는 자. 물이 돌을 뚫듯, 여러 번 시도해서 결국 바꾼다 |
| 10 | 마갈(磨羯) | 발효(醱酵) Fermentation | 변성자 | 내부에서 천천히 변질되는 자. 인내 끝의 변화 |
| 11 | 보병(寶甁) | 증식(增殖) Multiplication | 증폭자 | 하나에서 여럿으로 퍼지는 자. 전파와 혁신 |
| 12 | 쌍어(雙魚) | 투사(投射) Projection | 촉매자 | 닿은 것을 바꾸는 자. 자신은 변하지 않지만 곁의 것이 변한다 |

> 아키타입은 초안. 추후 다듬기 필요.

### 3 특수인 — 연금술 삼원질(Tria Prima)

| # | 인 이름 | 원리 | 성질 | 계승 특수 효과 |
|---|--------|------|------|-------------|
| 13 | 염(鹽) Salt | 삼원질 — 체(體) | 물질의 몸. 형태와 저항 | 소재 시 직접 계승 슬롯 +1 |
| 14 | 수은(水銀) Mercury | 삼원질 — 신(神) | 물질의 정신. 유동과 변환 | 소재 시 합성 발동 조건 완화 |
| 15 | 유황(硫黃) Sulfur | 삼원질 — 혼(魂) | 물질의 영혼. 연소와 의지 | 소재 시 잠재력 인하량 2배 |

특수인은 합체 테이블에 **정인과 동등하게 참여**한다 (15×15 테이블).
합체 결과의 인 결정은 정인과 같은 규칙을 따르되,
**트레잇 계승 3경로에서 특수 효과를 발휘**하는 것이 차별점.

플레이어가 일부러 삼원질 인의 유닛을 소재로 준비하는 전략적 동기:
- 염 소재: 넘기고 싶은 트레잇이 많을 때 → 직접 계승 슬롯 확보
- 수은 소재: 노리는 합성 트레잇의 조건이 까다로울 때 → 합성 문턱 낮춤
- 유황 소재: 다음 세대 합체를 위한 장기 투자 → 잠재력을 두텁게 깔기

## 합체 테이블 구조

### 기본 구조: 15×15 테이블

12정인 + 3특수인이 모두 동등하게 참여하는 15×15 테이블 (225칸, 대칭이면 120칸).
인A × 인B = 인C로 결과 인이 결정되고, 소재 레벨 평균으로 해당 인 내 구체 유닛이 결정된다.

```
인A(백양) × 인B(거해) = 인C(천갈) [테이블]
소재 레벨 평균 = (15 + 12) / 2 ≈ 14
→ 천갈 인 유닛 중 레벨 14 근처의 유닛이 결과물
→ 결과물의 트레잇은 3경로 계승 판정에 의해 결정
```

### 특수인이 소재일 때의 추가 효과

합체 결과(인C)의 결정은 테이블 규칙 그대로.
차이가 나는 건 **트레잇 계승 판정 단계**.

```
일반 합체: 정인A × 정인B
  → 합성 발동(통상) → 직접 계승(통상 슬롯) → 잠재력 계승(통상)

염 소재 합체: 염 유닛 × 정인B
  → 합성 발동(통상) → 직접 계승(슬롯 +1) → 잠재력 계승(통상)
  = 더 많은 트레잇을 직접 넘길 수 있음

수은 소재 합체: 수은 유닛 × 정인B
  → 합성 발동(조건 완화) → 직접 계승(통상) → 잠재력 계승(통상)
  = 보통이면 안 터지는 합성 조합이 발동할 수 있음

유황 소재 합체: 유황 유닛 × 정인B
  → 합성 발동(통상) → 직접 계승(통상) → 잠재력 계승(인하량 2배)
  = 잠재력이 두텁게 쌓여 다음 세대가 빠르게 성장
```

> 합체 테이블 구체 내용 (어떤 인×인이 어떤 인이 되는가)은 미확정.

## 인(印) × 조교 시스템

### 원칙: 포켓몬 성격치 방식

인이 조교 초기 경향의 **범위**를 정한다.
같은 인이라도 개체마다 구체적 성격은 다르지만, 인이 범위를 설정하고 개체가 그 안에서 랜덤.

## 트레잇 획득 시스템

### 경험치 풀 구조

하나의 경험치 풀이 **여러 트레잇 후보**를 담당한다.
행동으로 경험치가 쌓이면, 임계점 도달 후 확률 판정으로 풀 내 트레잇 중 하나가 해금된다.

```
[경험치 풀]    → [트레잇 후보군] (풀 내 랜덤)
전투 경험치    → 검술/창술/도술/화염마법/빙결마법/연속공격/...
신체 경험치    → 강건/민첩/야성/감각/...
성격 경험치    → 심약/호기심/냉정/반항적/순종적/...
성인 경험치    → 굴복/순종/수치심/신뢰/공포각인/... (상위는 도구로 접근 범위 결정)
```

> 경험치 풀의 구체적 분류와 풀별 트레잇 후보 목록은 미확정.

### 행동 → 경험치: 주+부 동시 획득

하나의 행동이 **주 경험치 + 부 경험치**를 동시에 올린다.

| 행동 | 주 경험치 | 부 경험치 |
|------|----------|----------|
| 전투 동행 | 전투 | 신체 |
| 훈련 | 전투 | — |
| 탐사 동행 | 신체 | 전투 |
| 교류 (대화/선물) | 성격 | — |
| 조교/교육 | 성인 | 성격 |
| 도시 배치 | (별도 검토) | — |

- 주 경험치: 해당 행동의 본래 목적. 높은 비율로 획득
- 부 경험치: 부수적으로 쌓임. 소량이지만 장기적으로 누적
- "전투만 시켜도 신체가 조금씩 오른다", "조교하면 성격도 조금 변한다"

### 해금 판정: 임계점 + 확률

```
경험치 < 임계점: 판정 자체가 발생하지 않음
경험치 ≥ 임계점: 매 관련 행동 시 확률 판정 발생
경험치가 더 쌓일수록: 판정 확률 상승
해금 시: 풀 내 후보 중 하나가 랜덤으로 열림
```

- 어떤 트레잇이 열릴지는 **확정이 아님** — 준비는 하되 결과에 랜덤성
- 이것이 다양한 조합을 만드는 핵심 장치 — 같은 유닛을 같은 방식으로 키워도 다른 트레잇이 열림
- 한번 해금된 트레잇은 풀에서 제거 → 다음 해금 시 나머지 후보 중에서 판정

### 해금 가중치를 주는 것

**인(印)**: 특정 풀의 경험치 획득 속도 + 풀 내 특정 트레잇 가중치
- 백양(선구자): 전투 풀 속도 UP + 풀 내 공격계 트레잇 가중치 UP

**도구**: 상위 트레잇 접근 허가 + 특정 트레잇 가중치
- Tier 1 트레잇: 도구 없이 해금 가능
- Tier 2+ 트레잇: 해당 도구가 있어야 후보 풀에 포함됨 (도구 = 잠금 해제 키)
- 예: 기초 조교도구 → ERA 기초 성인 트레잇 후보 개방
- 예: 상위 구속구 → 공포각인 계열 후보 개방

**동행 유닛 가중치: 기각** — 시스템 간 연쇄가 과도해지고, 하나의 트레잇으로 수렴하는 구조는 다양한 조합을 유도해야 하는 게임 방향과 맞지 않음.

### 미해금 경험치의 가치: 합체 잠재력 연결

경험치만 쌓이고 트레잇이 안 열린 상태에서 합체 소재로 쓰면:
- 해당 풀의 **임계점 인하**로 결과물에 잠재력 계승
- 경험치가 높을수록 인하량이 큼
- 결과물이 같은 풀에서 트레잇을 열 때 더 빠르게 판정 기회를 얻음

```
예시:
  소재 A: 전투 경험치 높음, 트레잇 미해금
  → 합체 → 결과물 C의 전투 풀 임계점 인하
  → C가 전투 경험치를 조금만 쌓아도 판정 기회 발생
  → 다음 세대에서 다시 시도, 빠르게 해금 가능
```

### 인(印)이 경험치에 미치는 영향

인은 **경험치 풀별 획득 배율**로 작용한다.

```
예시 (수치는 밸런싱 시 확정):
  백양(선구자):  전투 1.3x, 신체 1.1x, 성격 0.8x, 성인 0.8x
  거해(용해자):  전투 0.7x, 신체 0.9x, 성격 1.2x, 성인 1.3x
  천갈(추출자):  가장 높은 풀 1.5x, 나머지 0.7x (한 방향 극특화)
  쌍자(결박자):  전 풀 1.0x 균등, 대신 해금된 트레잇 안정성 높음
```

공정 원리가 경험치 프로필을 결정한다:
- 하소(태우기) → 전투/신체 빠름, 교류/조교 느림
- 용해(녹이기) → 성격/성인 빠름, 전투 느림
- 분리(떼어내기) → 하나에 집중, 나머지 느림
- 고착(고정하기) → 균등하지만 한번 열리면 안 변함
- 등등

### 인이 결정하지 않는 것

- 구체적 초기 성격 (개체별 랜덤, 인이 범위만 설정)
- 풀 내 어떤 트레잇이 열리는가 (랜덤, 인은 가중치만)
- 최종 트레잇 구성 (육성으로 얼마든지 변경 가능)
- 호감도 상한 (모든 유닛이 MAX 가능)

### 핵심: 인은 경사면이지 벽이 아니다

백양 인이라도 시간을 들이면 성인 트레잇을 키울 수 있다, 다만 거해보다 느릴 뿐.
이것이 만드는 분기:
- **효율 추구** → 인에 맞는 육성. 빠르고 예측 가능
- **애착 육성** → 인을 거슬러 육성. 느리지만 독특한 결과, 정이 듦

## 인(印) × 납품 시스템

### 원칙: 약하게 묶기 — 소폭 보정만

```
영혼력 = 기본 레벨값 + 트레잇 가산 + 육성 이력 보너스 + 인 보정
```

### 인 보정의 구성

**1. 인 기본 배율 (고정, 소폭)**

- 대부분의 정인: ×1.0 (보정 없음)
- 특수인 (염/수은/유황): ×1.15~1.2 (희소성 반영)

**2. 월별 수요 보정 (달력 연동)**

- 매달 2~3개 인의 수요가 상승
- 해당 인 납품 시 영혼력 추가 보정 (예: ×1.2)
- 플레이어가 "이번 달에 뭘 납품할까" 판단하는 근거
- 키워둔 유닛이 이번 달에 비싸면 "팔까?" = 감정적 판단 포인트

**3. 인-트레잇 일치 보너스**

- 유닛의 인에 어울리는 트레잇 → 추가 보정
  - 백양 인 + 전투 특화 트레잇 = "순수한 하소체" → 보너스
  - 백양 인 + 성인 특화 트레잇 = "변형된 하소체" → 보너스 없음
- 인에 맞게 키우면 납품가 상승
- 인을 거슬러 키우면 납품가는 낮지만 합체 소재로 독특한 가치

## 인(印) × 합체 소재 육성

### 유닛의 4가지 용도

| 용도 | 설명 | 인과의 관계 |
|------|------|-----------|
| 전력 | 파티에 넣고 탐사/전투 | 인에 맞는 육성 → 빠르게 강해짐 |
| 납품 | 팔아서 영혼력 획득 | 인-트레잇 일치 → 납품가 상승 |
| 소재 | 합체 재료로 사용 | 소재의 육성 방향이 결과물의 트레잇 계승을 결정 |
| 배치 | 도시 시설 운영 | (원소 속성이 주로 담당) |

### 소재 루트의 구조

합체는 역산이 핵심:

```
목표: 천갈 인 + "전투광" + "신뢰" 트레잇을 가진 유닛 C

역산:
  합체 테이블: 백양 × 거해 = 천갈

  소재 A(백양) → 전투광 트레잇 육성
    백양의 조교 가중치: 훈련 반응 높음 → 전투계 빠르게 성장
    인에 맞는 육성 = 효율적, 빨리 완성

  소재 B(거해) → 신뢰 트레잇 육성
    거해의 조교 가중치: 교류 반응 높음 → 관계계 빠르게 성장
    인에 맞는 육성 = 효율적

  합체 → C(천갈)에 전투광 + 신뢰 계승
```

### 소재 루트의 감정 비용 (ERA식 핵심)

"합치려고 키웠는데 정이 들었다"가 핵심 감정 루프.

인 체계가 이를 구조적으로 보장하는 방식:
- 인에 맞게 키울수록 **빨리 완성** → 정 들 시간이 적음
- 인을 거슬러 키울수록 **오래 걸림** → 정이 더 붙음
- 효율과 애착이 반비례

소재 육성 중에도 탐사 동행/이벤트/호감도 상승이 발생하므로,
"소재용으로 시작했지만 관계가 깊어진" 상황이 시스템적으로 발생한다.

### 육성 목적별 루트 정리

| 루트 | 인 활용 | 소요 시간 | 결과 | 감정 비용 |
|------|--------|----------|------|----------|
| 전력 + 인 순응 | 가중치 활용 | 짧음 | 예측 가능한 강한 유닛 | 낮음~중간 |
| 소재 + 인 순응 | 가중치 활용 | 짧음 | 효율적 소재 | 낮음 (빨리 끝남) |
| 소재 + 인 역행 | 가중치 역행 | 김 | 희귀한 트레잇 조합 소재 | 높음 (오래 키움) |
| 납품 + 시즌 | 월별 수요 활용 | 가변 | 영혼력 최적화 | 가변 (타이밍 압박) |

## 합체 시 트레잇 계승 시스템

### 레퍼런스

| 게임 | 계승 방식 | 이 게임에서 차용하는 것 |
|------|----------|---------------------|
| 메가텐 (SMT) | 소재의 스킬 중 일부를 결과물이 직접 계승. 슬롯 제한 | **직접 계승** — 기본 뼈대 |
| 우마무스메 | 부모의 육성 결과가 "인자"로 변환, 자식의 성장 보너스로 작용. 세대 누적 | **잠재력 계승** — 직접 안 넘어간 트레잇이 임계점 인하로 작용 |
| 자체 (조합 시스템 문법) | 태그 합산 → 특수 우선 → 미지 판정 | **합성 발동** — 서로 다른 소재의 트레잇이 합체 시점에 합성 트레잇으로 변환 |

메가텐의 직접 계승을 기본으로 하되, 우마무스메의 인자(잠재력 누적)와 자체 조합 시스템의 합성 발동을 결합한 **3경로 혼합** 구조.

### 3경로 구조

합체 시 소재 A/B의 트레잇은 3가지 경로 중 하나로 결과물 C에 전달된다.

```
소재 A의 트레잇들 + 소재 B의 트레잇들
          ↓ 합체 시 판정
  ┌───────┼───────┐
  ↓       ↓       ↓
합성 발동  직접 계승  잠재력 계승
(A+B→새것) (그대로)  (임계점↓)
```

**경로 1: 합성 발동 (최우선)**

소재 A의 트레잇과 B의 트레잇이 합성 조건을 만족하면, 직접 계승 대신 **합성 트레잇으로 변환**돼서 계승 슬롯에 우선 배치된다.

- 단일 육성에서도 합성 트레잇은 발현 가능하지만, 합체에서는 **서로 다른 유닛의 트레잇끼리** 합성 가능 → 단일 육성에서 불가능한 조합이 열림
- 합성에 사용된 원본 트레잇은 소멸 (합성 재료로 소비됨)
- 조합 시스템의 "특수 레시피 우선 참조"와 동일한 문법

```
예시:
  소재 A가 "심약" 보유 + 소재 B가 "수치심없음" 보유
  → 합성 조건 충족: 심약 + 수치심없음 → "유아퇴행" 합성 발동
  → 결과물 C에 "유아퇴행"이 계승 (심약, 수치심없음은 소멸)
```

**경로 2: 직접 계승 (남은 슬롯)**

합성에 쓰이지 않은 나머지 트레잇 중, 계승 슬롯 수만큼 선택돼서 결과물에 그대로 넘어간다.

- 메가텐의 스킬 계승과 동일한 구조
- 선택 기준: 성장도가 높은 순 (잘 키운 트레잇이 우선 계승)
- 계승 슬롯 수는 미확정 (종족별 차등? 인별 차등? 고정?)

**경로 3: 잠재력 계승 (나머지 전부)**

직접 계승 슬롯에서도 밀려난 트레잇. **버려지지 않고** 결과물의 해당 트레잇 획득 임계점을 낮춘다.

- 우마무스메의 인자(因子) 시스템과 동일한 원리
- "지금은 없지만 빨리 배울 수 있다" 상태로 결과물에 잠재
- 소재의 해당 트레잇 성장도에 비례하여 임계점 인하량 결정

```
예시:
  소재 A가 "화염 마법 Lv3" 보유, 직접 계승에서 밀려남
  → 결과물 C의 "화염 마법" 획득 임계점이 낮아짐
  → C가 훈련으로 화염 마법을 배울 때 필요한 경험치가 감소
```

### 판정 순서

```
합체 시 트레잇 판정 흐름:

1단계: 합성 체크 (최우선)
  A의 트레잇과 B의 트레잇 전체를 스캔
  합성 조건 충족 쌍이 있으면 → 합성 트레잇으로 변환, 계승 슬롯에 우선 배치
  합성에 사용된 원본 트레잇은 소멸
  ↓
2단계: 직접 계승 (남은 슬롯)
  합성에 쓰이지 않은 나머지 트레잇 중
  계승 슬롯 수만큼 선택 (성장도 높은 순)
  → 결과물에 그대로 계승
  ↓
3단계: 잠재력 계승 (나머지 전부)
  직접 계승에서도 밀려난 트레잇
  → 결과물의 해당 트레잇 임계점 인하로 전환
  → 소재의 성장도에 비례하여 인하량 결정
```

이 판정 흐름은 조합 시스템의 "태그 합산 → 특수 레시피 우선 → 자동 생성 → 미지" 판정과 동일한 문법 패턴을 따른다. 아이템이든 유닛이든 "재료를 합쳤을 때 결과를 판정하는 로직"이 같은 구조.

### 합체 체인의 가치: 잠재력 누적

합체를 반복할수록 잠재력이 누적된다.

```
[1세대]
  유닛 A (트레잇: 심약, 화염마법, 강건)
  유닛 B (트레잇: 수치심없음, 회복술)
  → 합체 → 유닛 C
    합성 발동: 심약+수치심없음 → 유아퇴행 (계승)
    직접 계승: 화염마법 (슬롯 1)
    잠재력 계승: 강건(임계점↓), 회복술(임계점↓)

[2세대]
  유닛 C를 육성: 잠재력 덕에 강건, 회복술을 빠르게 획득
  유닛 D (별도 육성)
  → 합체 → 유닛 E
    C에서 쌓인 트레잇 + D의 트레잇으로 더 복잡한 합성 가능
    1세대에서 잠재력으로 넘어온 것들이 2세대에서 활용됨

[3세대]
  ...세대가 쌓일수록 더 복잡한 트레잇 조합이 가능
```

이것이 합체 체인에 투자 동기를 만든다:
- 단발 합체: 즉각적 결과, 단순한 계승
- 다세대 합체: 시간 투자 필요, 하지만 단일 육성/단발 합체로 불가능한 트레잇 조합 도달 가능
- **세대를 쌓는다 = 잠재력을 축적한다 = 다음 세대의 가능성이 넓어진다**

### 3경로와 유닛 용도의 교차

| 용도 | 핵심 경로 | 판단 포인트 |
|------|----------|-----------|
| 전력 | 직접 계승 받은 트레잇으로 즉시 전투 투입 | 바로 쓸 수 있는 스킬이 넘어왔는가 |
| 소재 | 합성 발동을 노린 의도적 트레잇 배치 | A와 B에 어떤 트레잇을 키워야 합성이 터지는가 |
| 납품 | 인-트레잇 일치 보너스에 유리한 계승 | 인에 맞는 트레잇이 직접 계승됐는가 |
| 장기 투자 | 잠재력 누적으로 다음 세대 준비 | 이 유닛의 잠재력이 다음 합체에서 어떻게 쓰이는가 |

### 소재 설계의 깊이

3경로 구조가 만드는 **소재 설계 판단**:

합성 노리기 — "A에 심약, B에 수치심없음을 키워놓으면 합체 시 유아퇴행이 발동한다." 합성 레시피를 알고 있으면 의도적으로 소재를 설계하는 재미. 모르면 우연히 발견하는 재미.

계승 경쟁 — "직접 계승 슬롯이 3개인데 후보가 5개다. 어느 것을 넘기고 어느 것을 잠재력으로 보낼까?" 성장도를 의도적으로 조절해서 원하는 트레잇이 직접 계승되게 만드는 전략.

잠재력 활용 — "이번 세대에서는 직접 못 넘기지만, 잠재력으로 깔아놓으면 다음 세대에서 빠르게 키울 수 있다." 장기적 합체 체인 설계.

## 달력 연동 (느슨한 보정)

### 구조

1:1 고정 대응이 아니라, 매달 **활성 인 2~3개**가 순환.

```
예시:
  1월: 백양, 사자 활성 → 이 인의 유닛 조교 효율 UP, 납품 수요 UP
  2월: 거해, 천칭 활성
  ...
```

### 활성 인의 효과

- 해당 인 유닛의 조교 반응 전체에 소폭 보정 (예: 전 항목 +0.1x)
- 해당 인 유닛 납품 시 영혼력 추가 보정
- 해당 인의 유닛이 던전에서 조우 확률 상승 (?)
- 합체 시 해당 인이 결과로 나올 경우 보너스 (?)

> 달력 연동의 구체 효과와 수치는 미확정. 시즌 이벤트 체계(world_setting.md)와 통합 설계 필요.

## 특수인 3종의 특별한 위치

### 염(Salt) — 체(體)

- 조교 반응 전체적으로 느림
- **소재 시 직접 계승 슬롯 +1** — 넘기고 싶은 트레잇이 많을 때 핵심 소재
- "천천히 키워서 확실하게 물려주는" 소재형
- 납품가: 인 자체의 희소성으로 높음
- 플레이어 전략: 다양한 트레잇을 폭넓게 키운 뒤 소재로 투입

### 수은(Mercury) — 신(神)

- 조교 반응이 극도로 불안정 (같은 자극에 다른 반응 가능)
- **소재 시 합성 발동 조건 완화** — 보통이면 안 터지는 합성 조합이 발동 가능
- 도박형 소재 — 불안정한 육성 과정 자체가 예상 못한 트레잇을 만듦
- 납품가: 중간 (불안정성 때문에 평가 갈림)
- 플레이어 전략: 까다로운 합성 트레잇을 노릴 때 의도적으로 준비

### 유황(Sulfur) — 혼(魂)

- 조교 반응이 한 방향으로 극단적으로 쏠림
- 인-트레잇 일치 보너스가 다른 인의 2배
- **소재 시 잠재력 인하량 2배** — 다음 세대 합체를 위한 장기 투자
- "팔면 돈, 쓰면 장기 투자" 딜레마
- 납품가: 가장 높음
- 플레이어 전략: 합체 체인의 중간 세대에 넣어서 잠재력을 두텁게 깔기

## 원소 방어 프로필

### 구조: 상성이 아니라 6슬롯 방어 스탯

원소 간 상극/상생 관계는 **없다**. 유닛 개체마다 6종의 방어 슬롯을 가지며, 각 슬롯의 수치가 다를 뿐.

```
유닛 방어 프로필 (6슬롯):
  물리 방어  — 물리 공격에 대한 내성
  열 내성    — 열 속성 공격에 대한 내성
  위 내성    — 위 속성 공격에 대한 내성
  동 내성    — 동 속성 공격에 대한 내성
  광 내성    — 광 속성 공격에 대한 내성
  식 내성    — 식 속성 공격에 대한 내성
```

### 원칙

- **식(蝕) 내성에 특수 취급 없음** — 5원소 전부 동일한 규칙으로 작동
- 초기 방어 프로필은 대분류/개체에 따라 설정, 구체 수치는 밸런싱 시 확정
- **장비로 방어 프로필 변경 가능** — 열 내성이 낮은 유닛에게 열 내성 장비를 장착하면 보완됨
- 공격 속성은 장비가 결정 — "위 속성 무기로 공격하면 상대의 위 내성과 대조"하는 단순한 구조
- 원소 간 상호작용/변환 없음 — "열로 공격하면 동 내성이 떨어진다" 같은 연쇄 효과 없음

## 3축 분류의 역할 분담 요약

| 시스템 영역 | 담당 축 | 인(印)의 관여도 |
|------------|--------|---------------|

## 미확정 사항

### 인(印) 체계
- [ ] 12 정인 아키타입 다듬기 (현재 초안)
- [ ] 합체 테이블 구체 내용 (15×15 매트릭스)
- [ ] 조교 반응 가중치 구체 수치
- [ ] 트레잇 성장 가중치 구체 수치
- [ ] 트레잇 합성 친화 목록 (인별 완화 대상)
- [ ] 월별 활성 인 순환 테이블
- [ ] 달력 연동의 구체 효과/수치
- [ ] 인-트레잇 일치 보너스 판정 규칙
- [ ] 납품 보정 수치
- [ ] 대분류(세계관적 존재 유형)의 재설계 (확정: 요괴/정령/인조/야수/환상 + 추가 후보 검토 중. 방문자는 태그로 분리)
- [ ] 인의 세계관적 명칭/설정 ("인"이라고 부를 것인가, 다른 이름?)
- [ ] 전투 시 인이 관여하는 부분 (성격 트레잇 기반 AI와의 간섭 — 별도 검토 필요)
- [ ] 방어 프로필 초기값 기준 (대분류별? 개체별? 혼합?)

### 트레잇 획득 시스템
- [ ] 경험치 풀 구체 분류 (전투/신체/성격/성인 외 추가 풀 필요 여부)
- [ ] 풀별 트레잇 후보 목록 (각 풀에 어떤 트레잇이 들어가는가)
- [ ] 임계점 수치 및 확률 상승 곡선
- [ ] 주+부 경험치 비율 (주:부 = 3:1? 5:1?)
- [ ] 트레잇 Tier 구분 (Tier 1=도구 불필요, Tier 2+=도구 필요의 경계선)
- [ ] 도구별 개방 트레잇 범위 (어떤 도구가 어떤 후보를 풀에 추가하는가)
- [ ] 도시 배치 시 경험치 획득 여부/방식

### 트레잇 계승 시스템
- [ ] 계승 슬롯 수 (종족별 차등? 인별 차등? 고정?)
- [ ] 직접 계승 선택 기준 상세 (성장도 순? 플레이어 선택?)
- [ ] 합성 발동 레시피 구체 목록 (어떤 트레잇 조합이 어떤 합성 트레잇을 발동하는가)
- [ ] 잠재력 인하량 공식 (소재 성장도/미해금 경험치 → 임계점 감소량 변환)
- [ ] 잠재력 누적 상한 여부 (세대가 쌓일수록 무한 누적? 감쇠?)
- [ ] 합성 발동 시 계승 슬롯 소비 규칙 (합성 1건 = 슬롯 1개?)
- [ ] 특수인(염/수은/유황)이 소재일 때의 계승 특수 효과 수치 상세 (슬롯 +1은 확정? 합성 완화는 어느 정도?)

| 계승 판정 우선순위 | 합성 발동 → 직접 계승 → 잠재력 계승 | 조합 시스템 판정 문법과 동일 패턴 |
| 잠재력 누적 | 합체 체인(세대)을 거듭할수록 누적 | 합체 반복의 장기 투자 동기 |

| 경험치 구조 | 주+부 동시 획득 | 행동 1개가 복수 풀에 기여 |
| 도구 역할 | 상위 트레잇 접근 허가 (잠금 해제 키) | crafting 연결 |
| 동행 유닛 가중치 | 기각 | 시스템 연쇄 과도, 다양성 저해 |
| 아키타입 수정 | 쌍자→결박자, 천칭→극단자, 인마→침투자, 쌍어→촉매자 | 공정 원리에서 직접 파생 |

| 대분류 방향 | 기원×성질 기반, 방문자는 태그로 분리 | 요괴/정령/인조/야수/환상 확정, 나머지 검토 중 |
